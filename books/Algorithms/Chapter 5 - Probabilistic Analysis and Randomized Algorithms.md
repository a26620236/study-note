# Chapter 5: Probabilistic Analysis and Randomized Algorithms

**本章重點**：本章介紹機率分析（Probabilistic Analysis）和隨機演算法（Randomized Algorithms）的基本概念與應用。透過招聘問題作為範例，說明如何使用指示器隨機變數來簡化機率分析，並探討隨機演算法如何透過引入隨機性來改善平均效能。

## 5.1 招聘問題 (The Hiring Problem)

### 問題描述

假設你需要招聘一名辦公室助理，透過仲介公司每天面試一位候選人：
- **面試成本**：每次面試成本為 c_i（較低）
- **招聘成本**：每次招聘成本為 c_h（較高，c_h > c_i）
- **策略**：每當遇到比目前更優秀的候選人就立即招聘

**演算法 HIRE-ASSISTANT(n)**：
```
best = 0  // 假設的最低能力
for i = 1 to n
    面試候選人 i
    if 候選人 i 比當前最佳候選人更優秀
        best = i
        招聘候選人 i
```

### 成本分析

- **最壞情況**：候選人依能力遞增順序出現，招聘 n 次
  - 總成本：O(c_i × n + c_h × n)
- **最佳情況**：第一位候選人就是最優秀的，只招聘 1 次
  - 總成本：O(c_i × n + c_h)

### 機率分析的前提

為了進行機率分析，我們需要對輸入分佈做出假設：

**關鍵假設**：候選人以**隨機順序**到達
- 假設 n 位候選人可以用排名 1 到 n 表示（排名越高越優秀）
- 候選人的到達順序等機率地是 n! 種排列中的任一種
- 這種分佈稱為**均勻隨機排列** (uniform random permutation)

### 平均情況分析與隨機演算法的區別

**平均情況分析**：
- 對輸入分佈做出假設（如候選人隨機到達）
- 計算在這個分佈下的平均執行時間

**隨機演算法**：
- 演算法本身引入隨機性（如隨機排列輸入）
- 不需要假設輸入分佈
- 計算期望執行時間 (expected running time)

## 5.2 指示器隨機變數 (Indicator Random Variables)

### 定義

給定樣本空間 S 和事件 A，與事件 A 關聯的**指示器隨機變數** I{A} 定義為：

```
I{A} = { 1  如果 A 發生
       { 0  如果 A 不發生
```

### 核心引理

**Lemma 5.1**：給定事件 A，令 X_A = I{A}，則：

```
E[X_A] = Pr{A}
```

**意義**：指示器隨機變數的期望值等於事件發生的機率。

### 簡單例子：擲硬幣

投擲一枚公平硬幣，定義指示器隨機變數 X_H = I{正面}：
- X_H = 1（如果出現正面）
- X_H = 0（如果出現反面）

期望值：
```
E[X_H] = 1 × Pr{正面} + 0 × Pr{反面}
       = 1 × (1/2) + 0 × (1/2)
       = 1/2
```

### 投擲 n 次硬幣的分析

**問題**：投擲 n 次公平硬幣，期望出現多少次正面？

**傳統方法**：計算出現 0 次、1 次、2 次...正面的機率，然後求期望（複雜）

**使用指示器隨機變數**：
1. 定義 X_i = I{第 i 次投擲出現正面}，i = 1, 2, ..., n
2. 總正面數 X = X_1 + X_2 + ... + X_n
3. 根據**期望的線性性** (linearity of expectation)：

```
E[X] = E[X_1 + X_2 + ... + X_n]
     = E[X_1] + E[X_2] + ... + E[X_n]
     = 1/2 + 1/2 + ... + 1/2
     = n/2
```

**重要性質**：期望的線性性**即使在隨機變數相依的情況下也成立**。

### 招聘問題的分析

**目標**：計算期望招聘次數

**使用指示器隨機變數**：
1. 定義 X_i = I{候選人 i 被招聘}
2. 總招聘次數 X = X_1 + X_2 + ... + X_n

**計算 E[X_i]**：
- 候選人 i 被招聘當且僅當候選人 i 比前面所有候選人（1 到 i-1）都優秀
- 由於候選人隨機到達，前 i 位候選人中任何一位都有相同機率是最優秀的
- 因此：Pr{候選人 i 被招聘} = 1/i

**期望招聘次數**：
```
E[X] = E[X_1 + X_2 + ... + X_n]
     = E[X_1] + E[X_2] + ... + E[X_n]
     = 1/1 + 1/2 + 1/3 + ... + 1/n
     = ln n + O(1)  （調和級數）
```

**Lemma 5.2**：假設候選人以隨機順序到達，HIRE-ASSISTANT 的平均情況總招聘成本為 O(c_h ln n)。

這比最壞情況的 O(c_h n) 有顯著改善！

## 5.3 隨機演算法 (Randomized Algorithms)

### 動機

在許多情況下，我們無法確定或假設輸入分佈。此時可以使用**隨機演算法**：讓演算法本身引入隨機性。

### 隨機數產生器

假設我們有隨機數產生器 **RANDOM(a, b)**：
- 回傳介於 a 和 b 之間的整數（包含端點）
- 每個整數出現的機率相等：1/(b - a + 1)
- 每次呼叫的結果相互獨立

### 隨機化招聘演算法

**RANDOMIZED-HIRE-ASSISTANT(n)**：
```
1. 隨機排列候選人名單
2. 執行 HIRE-ASSISTANT(n)
```

**Lemma 5.3**：RANDOMIZED-HIRE-ASSISTANT 的期望招聘成本為 O(c_h ln n)。

**證明**：隨機排列輸入等同於假設輸入為隨機順序的機率分析情況。

**關鍵差異**：
- **機率分析**（Lemma 5.2）：假設輸入隨機
- **隨機演算法**（Lemma 5.3）：強制輸入隨機化，對任何輸入都有相同期望成本

### 隨機排列陣列

**目標**：產生**均勻隨機排列** (uniform random permutation)
- n 個元素有 n! 種排列
- 每種排列出現的機率應為 1/n!

**RANDOMLY-PERMUTE(A, n)**：
```
for i = 1 to n
    與 A[RANDOM(i, n)] 交換 A[i]
```

**工作原理**：
- 第 i 次迭代時，從 A[i] 到 A[n] 中隨機選一個元素與 A[i] 交換
- 保證每個元素在每個位置的機率都相等

### 正確性證明

**Lemma 5.4**：RANDOMLY-PERMUTE 產生均勻隨機排列。

**證明**（使用迴圈不變式）：
- **不變式**：在第 i 次迭代前，對於每個可能的 (i-1)-排列，子陣列 A[1..i-1] 包含這個排列的機率為 (n - i + 1)!/n!

**初始化**（i = 1）：
- A[1..0] 是空陣列
- 包含 0-排列的機率為 n!/n! = 1 ✓

**維護**：
- 假設不變式在第 i 次迭代前成立
- 第 i 次迭代從 n - i + 1 個元素中隨機選擇一個放入 A[i]
- 特定 i-排列出現的機率 = (前 i-1 個元素的機率) × (選中第 i 個元素的機率)
  = [(n - i + 1)!/n!] × [1/(n - i + 1)]
  = (n - i)!/n! ✓

**終止**（i = n + 1）：
- A[1..n] 包含特定 n-排列的機率為 0!/n! = 1/n! ✓

因此產生均勻隨機排列。

## 5.4 機率分析與指示器隨機變數的進一步應用

本節透過四個例子展示機率分析和指示器隨機變數的應用：
1. 生日悖論
2. 球與箱問題
3. 連續序列問題
4. 線上招聘問題

### 5.4.1 生日悖論 (The Birthday Paradox)

**問題**：房間裡需要多少人，才能使至少兩人生日相同的機率達到 50%？

**假設**：
- 一年有 n = 365 天
- 生日均勻分佈在 365 天
- 每個人的生日相互獨立

#### 方法一：直接機率計算

**策略**：計算補事件的機率（所有人生日都不同）

令 B_k = k 個人生日都不同的事件

**遞迴關係**：
```
Pr{B_k} = Pr{B_{k-1}} × Pr{第 k 個人的生日與前 k-1 人都不同 | B_{k-1}}
        = Pr{B_{k-1}} × (n - k + 1)/n
```

**展開**：
```
Pr{B_k} = 1 × (n-1)/n × (n-2)/n × ... × (n-k+1)/n
        = (1 - 1/n)(1 - 2/n)...(1 - (k-1)/n)
```

**上界估計**（使用 1 + x ≤ e^x）：
```
Pr{B_k} ≤ e^(-1/n) × e^(-2/n) × ... × e^(-(k-1)/n)
        = e^(-k(k-1)/(2n))
```

當 k(k-1)/(2n) ≥ ln(1/2) 時，Pr{B_k} ≤ 1/2

解得：k ≥ (1 + √(1 + 8ln2 × n))/2

**對於 n = 365**：k ≥ 23
- **結論**：23 個人就有超過 50% 的機率存在生日相同的配對！
- 火星（669 天）：需要 31 個火星人

#### 方法二：使用指示器隨機變數

定義指示器隨機變數 X_ij，對於每對 (i, j)，i < j：
```
X_ij = I{第 i 人和第 j 人生日相同}
```

E[X_ij] = Pr{兩人生日相同} = 1/n

總配對數 X = Σ_{i=1}^{k-1} Σ_{j=i+1}^k X_ij

期望配對數：
```
E[X] = Σ E[X_ij]
     = C(k, 2) × 1/n
     = k(k-1)/(2n)
```

當 k(k-1) ≥ 2n 時，E[X] ≥ 1，即期望至少有一對相同生日。

對於 n = 365：k ≥ √(2×365) + 1 ≈ 28
- 28 個人時，期望配對數 ≈ 1.0356

**兩種方法的比較**：
- 方法一：找機率 ≥ 1/2 的最小人數 → k = 23
- 方法二：找期望配對數 ≥ 1 的最小人數 → k = 28
- 兩者漸近相同：Θ(√n)

### 5.4.2 球與箱問題 (Balls and Bins)

將相同的球隨機投入 b 個箱子，每個球落入任何箱子的機率都是 1/b。

#### 三個問題

**1. 特定箱子中有多少球？**
- 投擲 n 個球，落入特定箱子的球數服從**二項分佈** b(n, 1/b)
- 期望球數：n/b

**2. 平均需要投擲多少球才能讓特定箱子有球？**
- 服從**幾何分佈**，成功機率為 1/b
- 期望投擲次數：b

**3. 需要投擲多少球才能讓每個箱子都至少有一球？**

**分析**（分階段）：
- 第 i 階段：從第 (i-1) 次命中到第 i 次命中
- 第 i 階段開始時，已有 i-1 個箱子有球，b-i+1 個箱子是空的
- 命中機率：(b - i + 1)/b
- 第 i 階段的期望投擲次數：b/(b - i + 1)

**總期望投擲次數**：
```
E[n] = Σ_{i=1}^b E[n_i]
     = Σ_{i=1}^b b/(b - i + 1)
     = b × Σ_{i=1}^b 1/i
     = b × (ln b + O(1))
```

**收集者問題** (Coupon Collector's Problem)：
- 如果你要收集 b 種不同的優惠券
- 期望需要獲得約 b ln b 張隨機優惠券才能集滿所有種類

### 5.4.3 連續序列 (Streaks)

**問題**：投擲公平硬幣 n 次，期望最長連續正面序列的長度是多少？

**答案**：Θ(lg n)

#### 上界證明：E[L] = O(lg n)

令 A_ik = 從第 i 次投擲開始出現至少 k 個連續正面的事件

**機率**：
```
Pr{A_ik} = 1/2^k
```

**對於 k = 2⌈lg n⌉**：
```
Pr{A_{i,2⌈lg n⌉}} ≤ 1/2^(2lg n) = 1/n²
```

至少一處出現長度 ≥ 2⌈lg n⌉ 連續正面的機率：
```
Pr{⋃_{i=1}^{n-2⌈lg n⌉+1} A_{i,2⌈lg n⌉}} ≤ Σ Pr{A_{i,2⌈lg n⌉}}  (Boole不等式)
                                        < n × 1/n²
                                        = 1/n
```

**期望計算**：
令 L_j = 最長序列恰好為 j 的事件，L = 最長序列長度

```
E[L] = Σ_{j=0}^n j × Pr{L_j}
     = Σ_{j=0}^{2⌈lg n⌉-1} j × Pr{L_j} + Σ_{j=2⌈lg n⌉}^n j × Pr{L_j}
     < 2⌈lg n⌉ × 1 + n × (1/n)
     = O(lg n)
```

#### 下界證明：E[L] = Ω(lg n)

**策略**：將 n 次投擲分成約 n/s 組，每組 s = ⌊(lg n)/2⌋ 次投擲

**單組全部為正面的機率**：
```
Pr{A_{i,⌊(lg n)/2⌋}} = 1/2^⌊(lg n)/2⌋ ≥ 1/√n
```

**所有組都失敗的機率**：
```
(1 - 1/√n)^⌊n/⌊(lg n)/2⌋⌋ ≤ (1 - 1/√n)^(2n/lg n - 1)
                           ≤ e^(-(2n/lg n - 1)/√n)
                           = O(e^(-ln n))
                           = O(1/n)
```

因此，至少有一組全為正面的機率 ≥ 1 - O(1/n)

**期望下界**：
```
E[L] ≥ 0 × Σ_{j=0}^{⌊(lg n)/2⌋-1} Pr{L_j} + ⌊(lg n)/2⌋ × Σ_{j=⌊(lg n)/2⌋}^n Pr{L_j}
     ≥ 0 + ⌊(lg n)/2⌋ × (1 - O(1/n))
     = Ω(lg n)
```

**結論**：最長連續正面序列的期望長度為 Θ(lg n)

#### 使用指示器隨機變數的近似分析

定義 X_ik = I{從第 i 次投擲開始至少 k 個連續正面}

期望長度 ≥ k 的序列數：
```
E[X_k] = Σ_{i=1}^{n-k+1} E[X_ik]
       = (n - k + 1)/2^k
```

**對於 k = c lg n**（c 為常數）：
```
E[X_{c lg n}] = (n - c lg n + 1)/2^(c lg n)
              = (n - c lg n + 1)/n^c
              = Θ(1/n^(c-1))
```

- 當 c 很大時，E[X_{c lg n}] 很小 → 不太可能出現長度 c lg n 的序列
- 當 c = 1/2 時，E[X_{(1/2) lg n}] = Θ(√n) → 很可能出現長度 (1/2) lg n 的序列

因此，最長序列期望長度為 Θ(lg n)。

### 5.4.4 線上招聘問題 (The Online Hiring Problem)

**問題變體**：
- 不想面試所有候選人來找到最佳人選
- 想避免多次招聘和解雇
- 願意接受接近最佳的候選人，但只招聘一次
- **規則**：面試後必須立即決定招聘或拒絕（不能回頭）

**策略**：
1. 選擇一個正整數 k < n
2. 面試並拒絕前 k 位候選人（觀察階段）
3. 之後招聘第一位分數高於前 k 位所有人的候選人
4. 如果沒有這樣的候選人，招聘最後一位（第 n 位）

**演算法 ONLINE-MAXIMUM(k, n)**：
```
best-score = -∞
for i = 1 to k
    if score(i) > best-score
        best-score = score(i)

for i = k+1 to n
    if score(i) > best-score
        return i  // 招聘候選人 i

return n  // 招聘最後一位候選人
```

#### 成功機率分析

令 S = 成功招聘到最佳候選人的事件
令 S_i = 當最佳候選人是第 i 位時成功的事件

```
Pr{S} = Σ_{i=k+1}^n Pr{S_i}
```

**計算 Pr{S_i}**：

要在最佳候選人是第 i 位時成功，需要：
1. 最佳候選人在位置 i：Pr{B_i} = 1/n
2. 位置 k+1 到 i-1 的候選人都不被選中，即前 i-1 位中的最大值在前 k 位：Pr{O_i} = k/(i-1)

由於 B_i 和 O_i 獨立：
```
Pr{S_i} = Pr{B_i} × Pr{O_i} = (1/n) × (k/(i-1)) = k/(n(i-1))
```

**總成功機率**：
```
Pr{S} = Σ_{i=k+1}^n k/(n(i-1))
      = (k/n) × Σ_{i=k}^{n-1} 1/i
```

**使用積分近似**：
```
∫_k^n (1/x)dx ≤ Σ_{i=k}^{n-1} 1/i ≤ ∫_{k-1}^{n-1} (1/x)dx

(k/n)(ln n - ln k) ≤ Pr{S} ≤ (k/n)(ln(n-1) - ln(k-1))
```

**最佳化 k 值**：

對下界 (k/n)(ln n - ln k) 微分：
```
d/dk [(k/n)(ln n - ln k)] = (1/n)(ln n - ln k - 1)
```

令導數為 0：
```
ln k = ln n - 1 = ln(n/e)
k = n/e
```

**結論**：當 k = n/e 時，成功招聘到最佳候選人的機率至少為 **1/e ≈ 0.368**

- 定義 X_ij = I{(i,j) 是逆序對}
- E[X_ij] = 1/2（隨機排列中，A[i] > A[j] 機率為 1/2）

## 章節總結

### 核心概念

1. **機率分析 vs 隨機演算法**
   - 機率分析：假設輸入分佈，計算平均情況效能
   - 隨機演算法：演算法本身引入隨機性，不依賴輸入分佈假設

2. **指示器隨機變數**
   - 將事件轉換為隨機變數：I{A} = 1 如果 A 發生，否則為 0
   - 核心性質：E[I{A}] = Pr{A}
   - 結合期望線性性，大幅簡化機率分析

3. **期望的線性性**
   - E[X + Y] = E[X] + E[Y]
   - **即使 X 和 Y 相依也成立**
   - 是機率分析中最強大的工具之一

### 重要結果

- **招聘問題**：期望招聘次數為 ln n（使用指示器隨機變數）
- **生日悖論**：23 人中有 50% 機率存在生日相同的配對（Θ(√n)）
- **收集者問題**：需要 b ln b 次嘗試才能收集到 b 種優惠券
- **連續正面序列**：n 次投擲期望最長連續正面為 Θ(lg n)
- **線上招聘**：使用 k = n/e 策略，至少有 1/e 機率選中最佳候選人

### 分析技巧

1. 使用指示器隨機變數分解複雜事件
2. 利用期望線性性簡化計算
3. 透過隨機化消除對輸入分佈的依賴
4. 使用補事件簡化機率計算
5. 階段分析法（如球與箱問題）
6. 上下界分析法（如連續序列問題）
