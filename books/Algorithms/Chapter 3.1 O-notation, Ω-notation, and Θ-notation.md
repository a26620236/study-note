## Chapter 3 介紹：表徵運行時間（Characterizing Running Times）

### 為什麼關注增長的階數（Order of Growth）？

演算法運行時間的**增長階數**提供了一個簡單的方法來：
1. **表徵演算法效率**
2. **比較不同演算法**

**關鍵觀察**：當輸入規模 n 足夠大時，精確運行時間中的**乘法常數**和**低階項**會被輸入規模本身的影響所主導。

**範例**：
- Merge Sort：Θ(n lg n) 最壞情況
- Insertion Sort：Θ(n²) 最壞情況
- 當 n 足夠大時，Merge Sort 總是勝出

### 漸近效率（Asymptotic Efficiency）

我們關注的是：**當輸入規模趨近無窮大時，運行時間如何增長**

- 通常情況下，**漸近上更有效率的演算法**是除了極小輸入外的最佳選擇
- 我們不需要計算精確的運行時間，只需要知道增長的速率

### 漸近記號的用途

本章介紹了幾種**標準化簡漸近分析的方法**：
- **Θ-notation**（緊界）
- **O-notation**（上界）
- **Ω-notation**（下界）

這些記號不僅用於分析**運行時間**，也可用於其他特性（例如空間使用量），甚至是**與演算法無關的一般函數**。

## 3.1 O-notation, Ω-notation, and Θ-notation

### 為什麼需要漸近記號？

回顧 Chapter 2 中 Insertion Sort 的精確運行時間：

```
T(n) = (c₅/2 + c₆/2 + c₇/2)n² + (c₁ + c₂ + c₄ + c₅/2 - c₆/2 - c₇/2 + c₈)n
       - (c₂ + c₄ + c₅ + c₈)
```

**簡化步驟**：
1. **捨棄低階項**：n 和常數項
2. **忽略最高階項的係數**：(c₅/2 + c₆/2 + c₇/2)
3. **只保留增長速率**：n²

結果：T(n) = **Θ(n²)**

---

### O-notation（大 O 記號）- 上界

#### 直觀定義

O-notation 表徵函數的**漸近上界**：
- 函數的增長速度**不快於**某個速率
- 基於**最高階項**

#### 範例

考慮函數：**f(n) = 7n³ + 100n² - 20n + 6**

- 最高階項：7n³
- 增長速率：n³

因此：
- ✅ f(n) = **O(n³)**（正確：增長不快於 n³）
- ✅ f(n) = **O(n⁴)**（也正確：增長速度確實慢於 n⁴）
- ✅ f(n) = **O(n⁵)**（也正確）
- ✅ f(n) = **O(n^c)** for any c ≥ 3

#### 關鍵理解

- O-notation 給出的是**鬆散的上界**
- 說一個 n³ 函數是 O(n⁴) 在技術上是正確的，但不夠精確
- 通常我們希望給出**盡可能緊的上界**

---

### Ω-notation（大 Omega 記號）- 下界

#### 直觀定義

Ω-notation 表徵函數的**漸近下界**：
- 函數的增長速度**至少和**某個速率一樣快
- 同樣基於**最高階項**

#### 範例

同樣的函數：**f(n) = 7n³ + 100n² - 20n + 6**

因此：
- ✅ f(n) = **Ω(n³)**（正確：增長至少和 n³ 一樣快）
- ✅ f(n) = **Ω(n²)**（也正確）
- ✅ f(n) = **Ω(n)**（也正確）
- ✅ f(n) = **Ω(n^c)** for any c ≤ 3

---

### Θ-notation（大 Theta 記號）- 緊界

#### 直觀定義

Θ-notation 表徵函數的**漸近緊界**：
- 函數**精確地**以某個速率增長
- 同時給出上界和下界（在常數倍數的範圍內）

#### 重要定理

如果可以證明函數同時是 **O(f(n)) 和 Ω(f(n))**，則該函數是 **Θ(f(n))**

#### 範例

函數：**f(n) = 7n³ + 100n² - 20n + 6**

- 已知：f(n) = O(n³)
- 已知：f(n) = Ω(n³)
- **結論**：f(n) = **Θ(n³)**

### 範例：Insertion Sort 的漸近分析

#### Insertion Sort 演算法

```
INSERTION-SORT(A, n)
1  for i = 2 to n
2      key = A[i]
3      // Insert A[i] into the sorted subarray A[1..i-1]
4      j = i - 1
5      while j > 0 and A[j] > key
6          A[j+1] = A[j]
7          j = j - 1
8      A[j+1] = key
```

#### 觀察與分析

**結構觀察**：
- **外層迴圈**（for）：執行 n-1 次，與輸入內容無關
- **內層迴圈**（while）：執行次數**取決於輸入數據**
  - 迴圈變數 j 從 i-1 開始遞減
  - 可能執行 0 次、i-1 次，或介於兩者之間
- **內層迴圈主體**（lines 6-7）：每次迭代花費**常數時間**

#### 證明 O(n²) 上界

**推論過程**：
1. 外層迴圈執行 n-1 次
2. 對於每次外層迴圈（i 值），內層最多執行 i-1 次
3. i 最大為 n
4. 總迭代次數最多：(n-1)(n-1) < n²
5. 每次迭代花費常數時間 c

**結論**：T(n) ≤ c·n² → **T(n) = O(n²)**（適用於所有輸入）

#### 證明 Ω(n²) 下界（最壞情況）

**目標**：證明存在某些輸入使運行時間至少為 cn²

**構造最壞情況輸入**：

假設 n 是 3 的倍數，將陣列分為三部分：
- **前 n/3 個位置**：A[1..n/3]
- **中間 n/3 個位置**：A[n/3+1..2n/3]
- **後 n/3 個位置**：A[2n/3+1..n]

**最壞情況**：前 n/3 個位置包含**最大的 n/3 個值**

**分析**：
1. 排序後，這 n/3 個最大值必須移動到後 n/3 個位置
2. **每個值**必須經過中間 n/3 個位置
3. line 6（移動元素）執行次數：至少 (n/3) × (n/3) = n²/9
4. 這是 Ω(n²)

**關鍵洞察**：
- 值要向右移動 k 個位置，line 6 必須執行 k 次
- 最大的 n/3 個值各自至少移動 n/3 次

**結論**：最壞情況下 **T(n) = Ω(n²)**

#### 最終結論

Insertion Sort 的最壞情況運行時間：
- 上界：O(n²)
- 下界：Ω(n²)
- **緊界**：**Θ(n²)**

## 重要概念總結

### 三種記號的關係

| 記號 | 含義 | 類比 | 範例（7n³ + 100n² - 20n + 6） |
|------|------|------|-------------------------------|
| **O** | 上界（≤） | "不超過" | O(n³), O(n⁴), O(n⁵) |
| **Ω** | 下界（≥） | "至少" | Ω(n³), Ω(n²), Ω(n) |
| **Θ** | 緊界（=） | "恰好" | Θ(n³) |

### 使用原則

1. **O-notation**：
   - 分析**任意情況**的上界
   - 給出"保證不會更慢"的承諾

2. **Ω-notation**：
   - 分析**最壞情況**的下界
   - 證明"某些輸入至少需要這麼長時間"

3. **Θ-notation**：
   - 同時證明 O 和 Ω
   - 給出**精確的增長速率**

### 分析技巧

1. **識別主導項**：找出最高階項
2. **忽略係數**：只關注增長速率
3. **捨棄低階項**：它們在 n→∞ 時可忽略
4. **構造最壞情況**：證明 Ω 界時，設計特定輸入使運行時間最長
