## 插入排序概述

### 問題定義

**排序問題**：
- **輸入**：n 個數字的序列 ⟨a₁, a₂, ..., aₙ⟩
- **輸出**：輸入序列的排列 ⟨a'₁, a'₂, ..., a'ₙ⟩，使得 a'₁ ≤ a'₂ ≤ ... ≤ a'ₙ

### 重要術語

**鍵（Keys）與衛星資料（Satellite Data）**：
- **鍵**：要排序的數字
- **衛星資料**：與鍵關聯的其他資料
- **記錄（Record）**：鍵和衛星資料一起形成記錄

**範例**：
- 學生記錄的試算表包含多個相關資料：年齡、平均成績、修課數量
- 這些數量中的任何一個都可以是鍵
- 當試算表排序時，它會將相關記錄（衛星資料）與鍵一起移動

**說明演算法時**：
- 我們專注於鍵
- 但要記住通常有相關的衛星資料

## 插入排序的工作原理

### 撲克牌類比

插入排序的工作方式就像你整理一手撲克牌：

1. **初始狀態**：
   - 左手空著
   - 牌堆放在桌上

2. **過程**：
   - 從牌堆中拿起第一張牌，放在左手
   - 然後用右手一次從牌堆中移除一張牌
   - 將其插入左手中的正確位置

3. **尋找正確位置**：
   - 將卡片與左手中已有的每張卡片比較
   - 從右邊開始向左移動
   - 一旦看到左手中值小於或等於右手卡片的卡片，就將右手的卡片插入該卡片的右側
   - 如果左手中所有卡片的值都大於右手的卡片，則將此卡片放在左手最左邊

4. **不變性質**：
   - 左手中的卡片始終是排序的
   - 這些卡片最初是桌上牌堆的頂部卡片

## 插入排序的偽代碼

### 演算法實現

```
INSERTION-SORT(A, n)
1  for i = 2 to n
2      key = A[i]
3      // Insert A[i] into the sorted subarray A[1:i-1].
4      j = i - 1
5      while j > 0 and A[j] > key
6          A[j+1] = A[j]
7          j = j - 1
8      A[j+1] = key
```

### 參數說明

- **A**：包含要排序的值的陣列
- **n**：要排序的值的數量
- **值的位置**：佔據陣列的位置 A[1] 到 A[n]，記為 A[1:n]

### 結果

當 INSERTION-SORT 過程完成時，陣列 A[1:n] 包含原始值，但按排序順序排列。

## 演算法運作範例

### 執行過程

考慮陣列 A 最初包含序列 ⟨5, 2, 4, 6, 1, 3⟩：

```
初始狀態：[5, 2, 4, 6, 1, 3]

迭代 1 (i=2):  [2, 5, 4, 6, 1, 3]  // 插入 2
迭代 2 (i=3):  [2, 4, 5, 6, 1, 3]  // 插入 4
迭代 3 (i=4):  [2, 4, 5, 6, 1, 3]  // 6 已在正確位置
迭代 4 (i=5):  [1, 2, 4, 5, 6, 3]  // 插入 1
迭代 5 (i=6):  [1, 2, 3, 4, 5, 6]  // 插入 3

最終結果：[1, 2, 3, 4, 5, 6]
```

### 關鍵觀察

- **索引 i**：指示當前被插入的「卡片」
- **子陣列 A[1:i-1]**：在每次 for 循環迭代開始時
  - 構成當前已排序的手牌
  - 包含原本在位置 1 到 i-1 的元素，但現在按排序順序排列
- **子陣列 A[i+1:n]**：對應桌上仍然存在的牌堆

- 棕褐色矩形：左側用於比較的值

- 藍色箭頭：鍵移動到的位置

## 循環不變式與正確性證明

### 什麼是循環不變式？

**循環不變式（Loop Invariant）**：描述循環每次迭代開始時必須為真的性質

**插入排序的循環不變式**：
> 在 for 循環（第 1-8 行）每次迭代開始時，子陣列 A[1:i-1] 包含原本在 A[1:i-1] 中的元素，但按排序順序排列。

### 證明正確性的三個性質

使用循環不變式證明演算法正確性時，需要證明三件事：

#### 1. 初始化（Initialization）

**證明**：在第一次循環迭代之前，循環不變式為真

- 當 i = 2 時，第一次循環迭代開始之前
- 子陣列 A[1:i-1] 僅由單個元素 A[1] 組成
- 這個元素就是 A[1] 中的原始元素
- 而且，這個子陣列已排序（只有一個值的子陣列怎麼可能不排序？）
- **結論**：循環不變式在第一次迭代之前成立

#### 2. 維護（Maintenance）

**證明**：如果循環不變式在某次迭代之前為真，則在下次迭代之前仍然為真

**非正式分析**：
- for 循環的主體通過以下方式工作：
  - 將 A[i-1]、A[i-2]、A[i-3] 等的值向右移動一個位置
  - 直到找到 A[i] 的適當位置（第 4-7 行）
  - 在該位置插入 A[i] 的值（第 8 行）
- 子陣列 A[1:i] 然後包含原本在 A[1:i] 中的元素，但按排序順序排列
- 為下一次迭代增加 i（將其值增加 1）後，保持了循環不變式

**正式分析**：
- 更正式的處理需要陳述並證明 while 循環（第 5-7 行）的循環不變式
- 這裡我們依靠非正式分析來證明外部循環的第二個性質成立

#### 3. 終止（Termination）

**證明**：當循環終止時，不變式提供有用的性質來幫助證明演算法正確

- 循環變數 i 從 2 開始，每次迭代增加 1
- 一旦 i 的值在第 1 行超過 n，循環終止
- 即，當 i 等於 n + 1 時循環終止
- 將 n + 1 代入循環不變式中的 i，得到：
  - 子陣列 A[1:n] 包含原本在 A[1:n] 中的元素，但按排序順序排列
- **因此，演算法是正確的**

### 循環不變式的重要性

**類比數學歸納法**：
- **基礎情況（Base Case）**：證明不變式在第一次迭代之前成立
- **歸納步驟（Inductive Step）**：證明不變式從一次迭代保持到下一次迭代
- **差異**：數學歸納法通常無限應用歸納步驟，但在循環不變式中，當循環終止時「歸納」停止

**第三個性質（終止）的重要性**：
- 這可能是最重要的性質
- 因為你使用循環不變式來證明正確性
- 通常使用循環不變式與導致循環終止的條件一起使用

## 偽代碼慣例

本書使用以下偽代碼慣例：

### 1. 縮排（Indentation）

- **縮排表示區塊結構**
- 範例：
  - for 循環（第 1 行開始）的主體包含第 2-8 行
  - while 循環（第 5 行開始）的主體包含第 6-7 行，但不包括第 8 行
- 縮排樣式也適用於 if-else 陳述句
- 使用縮排而不是 begin/end 或大括號減少混亂，同時保持或增強清晰度

### 2. 循環結構

**for、while、repeat-until 循環**：
- 解釋類似於 C、C++、Java、Python 和 JavaScript
- **重要差異**：循環計數器在循環退出後保留其值
  - 與某些 C++ 和 Java 情況不同
  - 循環退出後，循環計數器的值是首次超過 for 循環界限的值
  - 範例：for 循環標頭 `for i = 2 to n` 終止時，i = n + 1

**關鍵字**：
- **to**：for 循環在每次迭代中遞增其循環計數器
- **downto**：for 循環遞減其循環計數器（每次迭代減少 1）
- **by**：當循環計數器變化量大於 1 時，變化量跟在可選關鍵字 by 之後

### 3. 註釋

- 符號 `//` 表示該行的其餘部分是註釋

### 4. 變數

- 變數（如 i、j、key）對於給定的過程是局部的
- 我們不使用全域變數，除非明確指出

### 5. 陣列存取

**索引方式**：
- 通過指定陣列名稱後跟方括號中的索引來存取陣列元素
- 範例：A[i] 表示陣列 A 的第 i 個元素

**索引起點**：
- 許多程式語言強制使用 0 起源索引（0 是最小有效索引）
- 本書選擇對人類讀者最清晰的索引方案
- 由於人們通常從 1 開始計數，本書的大多數（但不是全部）陣列使用 1 起源索引
- 為了清楚起見，我們會明確指定陣列的邊界

**實現建議**：
- 如果你在強制 0 起源索引的語言（如 C、C++、Java、Python、JavaScript）中實現演算法
- 可以選擇：
  1. 始終從每個索引中減去 1
  2. 為每個陣列分配一個額外的位置，並忽略位置 0

### 6. 子陣列表示法

**符號 `:`**：
- 表示子陣列
- A[i:j] 表示 A 的子陣列，由元素 A[i]、A[i+1]、...、A[j] 組成
- 也用於表示陣列的邊界，如 A[1:n]

**注意**（Python 使用者）：
- 在本書中，子陣列 A[i:j] **包含**元素 A[j]
- 在 Python 中，A[i:j] 的最後一個元素是 A[j-1]
- Python 允許負索引（從後端計數），但本書不使用負陣列索引

### 7. 物件與屬性

**複合資料組織**：
- 將複合資料組織成物件，由屬性組成
- 使用物件導向程式語言中的語法存取特定屬性：
  - 物件名稱，後跟點，後跟屬性名稱
  - 範例：`x.f` 表示物件 x 的屬性 f

**物件與陣列的處理**：
- 將代表陣列或物件的變數視為指向資料的指標（某些語言稱為引用）
- 對於物件 x 的所有屬性 f，設定 `y = x` 會導致 `y.f` 等於 `x.f`
- 如果我們現在設定 `x.f = 3`，那麼之後不僅 `x.f` 等於 3，`y.f` 也等於 3
- 換句話說，賦值 `y = x` 之後，x 和 y 指向同一個物件
- 這種處理陣列和物件的方式與大多數當代程式語言一致

**屬性級聯**：
- 屬性表示法可以「級聯」
- 範例：假設屬性 f 本身是指向某種類型物件的指標，該物件具有屬性 g
- 則表示法 `x.f.g` 隱式括號化為 `(x.f).g`
- 如果我們賦值了 `y = x.f`，則 `x.f.g` 與 `y.g` 相同

**NIL 值**：
- 有時指標不指向任何物件
- 在這種情況下，我們給它特殊值 NIL

### 8. 參數傳遞

**按值傳遞**：
- 我們按值將參數傳遞給過程
- 被調用的過程接收參數的自己的副本
- 如果它為參數賦值，調用過程不會看到變化

**物件與陣列的特殊情況**：
- 當傳遞物件時，指向表示物件的資料的指標被複製，但物件的屬性不被複製
- 範例：
  - 如果 x 是被調用過程的參數
  - 被調用過程中的賦值 `x = y` 對調用過程不可見
  - 但賦值 `x.f = 3` 如果調用過程有指向與 x 相同物件的指標，則可見
- 陣列按指標傳遞：
  - 傳遞指向陣列的指標，而不是整個陣列
  - 對單個陣列元素的更改對調用過程可見
- 大多數當代程式語言都以這種方式工作

### 9. 返回陳述句

**return 陳述句**：
- 立即將控制權轉移回調用過程中的調用點
- 大多數 return 陳述句還取一個值傳回給調用者
- **本書的差異**：允許在單個 return 陳述句中返回多個值，而無需創建物件來打包它們

### 10. 布林運算子

**短路求值（Short-Circuiting）**：
- 布林運算子 `and` 和 `or` 是短路的

**and 運算子**：
- 計算表達式 `x and y` 時，首先計算 x
- 如果 x 計算為 FALSE，則整個表達式不能計算為 TRUE，因此不計算 y
- 如果 x 計算為 TRUE，則必須計算 y 以確定整個表達式的值

**or 運算子**：
- 在表達式 `x or y` 中，僅當 x 計算為 FALSE 時才計算 y

**優點**：
- 短路運算子允許我們編寫布林表達式，如 `x ≠ NIL and x.f = y`
- 無需擔心當 x 為 NIL 時計算 x.f 會發生什麼

### 11. 錯誤處理

**error 關鍵字**：
- 表示發生錯誤，因為調用過程的條件不正確
- 過程立即終止
- 調用過程負責處理錯誤
- 我們不指定要採取什麼行動
