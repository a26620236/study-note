## 2.3.1 分治法

### 基本概念

**遞迴結構**：
- 許多有用的演算法在結構上是遞迴的
- 為了解決給定問題，它們遞迴（呼叫自己）一次或多次
- 處理密切相關的子問題

**分治法的特徵**：
- 將問題分解成幾個子問題
- 子問題與原問題相似，但規模更小
- 遞迴地解決子問題
- 然後組合這些解來創建原問題的解

### 三個特徵步驟

分治法包含三個步驟：

#### 1. 分解（Divide）
將問題分解成一個或多個子問題，這些子問題是同一問題的較小實例

#### 2. 征服（Conquer）
通過遞迴求解子問題來征服它們

#### 3. 組合（Combine）
組合子問題的解以形成原問題的解

### 基礎情況與遞迴情況

**基礎情況（Base Case）**：
- 如果問題足夠小
- 直接解決它而不遞迴

**遞迴情況（Recursive Case）**：
- 否則，執行三個特徵步驟

## 合併排序（Merge Sort）

### 演算法概述

合併排序密切遵循分治法。在每一步中，它排序一個子陣列 A[p:r]：
- 從整個陣列 A[1:n] 開始
- 遞迴到越來越小的子陣列

### 合併排序的三個步驟

#### 1. 分解（Divide）

**操作**：
- 將要排序的子陣列 A[p:r] 分成兩個相鄰的子陣列
- 每個大約一半大小

**方法**：
- 計算 A[p:r] 的中點 q（取 p 和 r 的平均值）
- 將 A[p:r] 分成子陣列 A[p:q] 和 A[q+1:r]

#### 2. 征服（Conquer）

**操作**：
- 使用合併排序遞迴地排序兩個子陣列 A[p:q] 和 A[q+1:r]

#### 3. 組合（Combine）

**操作**：
- 將兩個已排序的子陣列 A[p:q] 和 A[q+1:r] 合併回 A[p:r]
- 產生排序後的答案

### 基礎情況

**遞迴終止**：
- 當要排序的子陣列 A[p:r] 只有 1 個元素時
- 即當 p 等於 r 時
- **單個元素的子陣列總是已排序的**

## MERGE 程序

### 撲克牌類比

**理解合併操作**：

想像你在桌上有兩堆正面朝上的牌：
1. **初始狀態**：
   - 每堆都已排序
   - 最小值的牌在頂部

2. **目標**：
   - 將兩堆合併成一個排序的輸出堆
   - 輸出堆正面朝下放在桌上

3. **基本步驟**：
   - 選擇兩堆頂部牌中較小的一張
   - 從其堆中移除它（露出新的頂部牌）
   - 將此牌正面朝下放在輸出堆上
   - 重複此步驟，直到一個輸入堆為空
   - 然後將剩餘的輸入堆翻轉並放在輸出堆上

### 時間複雜度分析

**基本步驟的成本**：
- 每個基本步驟需要常數時間
- 因為只比較兩張頂部牌

**總步驟數**：
- 如果兩個已排序的堆各有 n/2 張牌
- 基本步驟數至少為 n/2
- 至多為 n（實際上至多 n-1）
- 每個基本步驟需要常數時間
- 總步驟數在 n/2 和 n 之間

**結論**：
- 合併需要的時間大致與 n 成正比
- 即合併需要 **Θ(n) 時間**

### MERGE 程序的實現

```
MERGE(A, p, q, r)
1   nₗ = q - p + 1                    // A[p:q] 的長度
2   nᵣ = r - q                        // A[q+1:r] 的長度
3   let L[0:nₗ-1] and R[0:nᵣ-1] be new arrays
4   for i = 0 to nₗ - 1                // 將 A[p:q] 複製到 L[0:nₗ-1]
5       L[i] = A[p + i]
6   for j = 0 to nᵣ - 1                // 將 A[q+1:r] 複製到 R[0:nᵣ-1]
7       R[j] = A[q + j + 1]
8   i = 0                              // i 索引 L 中最小的剩餘元素
9   j = 0                              // j 索引 R 中最小的剩餘元素
10  k = p                              // k 索引 A 中要填充的位置
11  // 只要 L 和 R 都包含未合併的元素，
    // 將最小的未合併元素複製回 A[p:r]
12  while i < nₗ and j < nᵣ
13      if L[i] ≤ R[j]
14          A[k] = L[i]
15          i = i + 1
16      else A[k] = R[j]
17          j = j + 1
18      k = k + 1
19  // 已經完全遍歷了 L 或 R 中的一個，
    // 將另一個的剩餘部分複製到 A[p:r] 的末尾
20  while i < nₗ
21      A[k] = L[i]
22      i = i + 1
23      k = k + 1
24  while j < nᵣ
25      A[k] = R[j]
26      j = j + 1
27      k = k + 1
```

### MERGE 程序的詳細說明

**第 1-3 行：初始化**
- 計算 nₗ 和 nᵣ（兩個子陣列的長度）
- 創建臨時陣列 L[0:nₗ-1] 和 R[0:nᵣ-1]

**注意**：
- 這個程序罕見地同時使用 1 起源索引（陣列 A）和 0 起源索引（陣列 L 和 R）
- 對 L 和 R 使用 0 起源索引使循環不變式更簡單（見練習 2.3-3）

**第 4-7 行：複製資料**
- for 循環將子陣列 A[p:q] 複製到 L
- for 循環將子陣列 A[q+1:r] 複製到 R

**第 8-18 行：執行基本步驟**
- while 循環反覆識別 L 和 R 中尚未複製回的最小值並複製它
- 索引 k 給出正在填充的 A 的位置
- 索引 i 和 j 分別給出 L 和 R 中最小剩餘值的位置
- 最終，L 或 R 的所有內容都被複製回 A[p:r]，此循環終止

**第 20-27 行：複製剩餘元素**
- 如果循環因為 R 的所有內容已複製回而終止（j = nᵣ）
  - i 仍小於 nₗ，所以 L 的一些內容尚未複製回
  - 這些值是 L 和 R 中最大的
  - while 循環（第 20-23 行）將 L 的剩餘值複製到 A[p:r] 的最後幾個位置
  - 因為 j = nᵣ，while 循環（第 24-27 行）迭代 0 次
- 如果循環因為 i = nₗ 而終止
  - L 的所有內容已經複製回 A[p:r]
  - while 循環（第 24-27 行）將 R 的剩餘值複製回 A[p:r] 的末尾

### MERGE 程序的運行時間分析

**目標**：證明 MERGE 程序在 Θ(n) 時間內運行，其中 n = r - p + 1

**分析**：

1. **第 1-3 行和第 8-10 行**：
   - 每行需要常數時間

2. **第 4-7 行的 for 循環**：
   - 需要 Θ(nₗ + nᵣ) = Θ(n) 時間

3. **三個 while 循環（第 12-18、20-23、24-27 行）**：
   - 觀察：這些循環的每次迭代都將恰好一個值從 L 或 R 複製回 A
   - 每個值恰好被複製回 A 一次
   - 因此，這三個循環總共進行 n 次迭代
   - 由於每個循環的每次迭代需要常數時間
   - 這三個循環花費的總時間為 Θ(n)

**結論**：MERGE 程序在 Θ(n) 時間內運行

### 圖 2.3 說明

圖 2.3 展示了在呼叫 MERGE(A, 9, 12, 16) 時第 8-18 行 while 循環的操作：
- 子陣列 A[9:16] 包含值 ⟨2, 4, 6, 7, 1, 2, 3, 5⟩
- 分配並複製後：
  - 陣列 L 包含 ⟨2, 4, 6, 7⟩
  - 陣列 R 包含 ⟨1, 2, 3, 5⟩

**顏色標記**：
- **棕褐色位置**：A 中包含最終值；L 和 R 中包含尚未複製回 A 的值
- **藍色位置**：A 中將被覆蓋的值
- **深色位置**：L 和 R 中已經複製回 A 的值

## MERGE-SORT 程序

### 程序實現

```
MERGE-SORT(A, p, r)
1  if p ≥ r                    // 零個或一個元素？
2      return
3  q = ⌊(p + r)/2⌋             // A[p:r] 的中點
4  MERGE-SORT(A, p, q)         // 遞迴排序 A[p:q]
5  MERGE-SORT(A, q+1, r)       // 遞迴排序 A[q+1:r]
6  // 將 A[p:q] 和 A[q+1:r] 合併到 A[p:r]
7  MERGE(A, p, q, r)
```

### 程序說明

**參數**：
- A：陣列
- p, r：索引

**功能**：
- 排序子陣列 A[p:r] 中的元素

**基礎情況**：
- 如果 p = r，子陣列只有 1 個元素，因此已經排序
- 過程立即返回

**遞迴情況**：
- 如果 p < r，MERGE-SORT 運行分解、征服和組合步驟

**第 3 行（分解步驟）**：
- 簡單地計算將 A[p:r] 分成兩個相鄰子陣列的索引 q
- A[p:q] 包含 ⌈n/2⌉ 個元素
- A[q+1:r] 包含 ⌊n/2⌋ 個元素

**第 4-5 行（征服步驟）**：
- 遞迴地排序兩個子陣列

**第 7 行（組合步驟）**：
- 使用 MERGE 程序合併兩個已排序的子陣列

**初始呼叫**：
- MERGE-SORT(A, 1, n) 排序整個陣列 A[1:n]

### 向下取整和向上取整

**符號說明**：
- **⌈x⌉**：大於或等於 x 的最小整數（向上取整）
- **⌊x⌋**：小於或等於 x 的最大整數（向下取整）

**驗證子陣列大小**：
- 設定 q 為 ⌊(p + r)/2⌋ 產生：
  - 子陣列 A[p:q] 的大小為 ⌈n/2⌉
  - 子陣列 A[q+1:r] 的大小為 ⌊n/2⌋
- 驗證方法：檢查 p 和 r 是奇數還是偶數的四種情況

### 圖 2.4 說明

圖 2.4 展示了 n = 8 時合併排序的操作：
- 陣列 A 最初包含序列 ⟨12, 3, 7, 9, 14, 6, 11, 2⟩
- 演算法遞迴地將陣列分解到 1 元素子陣列
- 組合步驟合併：
  - 1 元素子陣列對形成長度為 2 的已排序子陣列
  - 再合併形成長度為 4 的已排序子陣列
  - 再合併形成長度為 8 的最終已排序子陣列

**非 2 的冪次的情況**：
- 如果 n 不是 2 的精確次方
- 某些分解步驟創建長度相差 1 的子陣列
- 例如：分解長度為 7 的子陣列時，一個子陣列長度為 4，另一個長度為 3
- 無論合併的兩個子陣列的長度如何
- **合併總共 n 個項目的時間為 Θ(n)**

## 2.3.2 分析分治演算法

### 遞迴方程

**當演算法包含遞迴呼叫時**：
- 通常可以用遞迴方程或遞迴來描述其運行時間
- 遞迴描述大小為 n 的問題的總運行時間
- 用同一演算法在較小輸入上的運行時間來表示
- 然後可以使用數學工具來求解遞迴
- 提供演算法性能的界限

### 分治遞迴的結構

**來源**：
- 分治演算法的運行時間遞迴來自基本方法的三個步驟

**記號**：
- 讓 T(n) 為大小為 n 的問題的最壞情況運行時間

**基礎情況**：
- 如果問題大小足夠小，比如 n < n₀（對於某個常數 n₀ > 0）
- 直接解決方案需要常數時間
- 我們寫作 **Θ(1)**

**遞迴情況**：
假設問題的分解產生 a 個子問題，每個大小為 n/b（即原始大小的 1/b）：
- **a**：子問題數量
- **b**：每個子問題的縮小因子
- 對於合併排序，a = 2 且 b = 2

**時間成本**：
- 解決一個大小為 n/b 的子問題需要 T(n/b) 時間
- 解決所有 a 個子問題需要 aT(n/b) 時間
- 如果分解問題需要 D(n) 時間
- 組合子問題的解需要 C(n) 時間

**遞迴方程**：

```
T(n) = { Θ(1)                    if n < n₀,
       { D(n) + aT(n/b) + C(n)   otherwise.
```

**Chapter 4**：展示如何求解這種形式的常見遞迴

### 處理向下取整和向上取整

**問題**：
- 有時 n/b 大小的分解步驟不是整數
- 例如：MERGE-SORT 將大小為 n 的問題分成大小為 ⌈n/2⌉ 和 ⌊n/2⌋ 的子問題
- ⌈n/2⌉ 和 ⌊n/2⌋ 之間的差異最多為 1

**簡化**：
- 對於大的 n，這個差異遠小於將 n 除以 2 的效果
- 我們「瞇著眼睛」將它們都稱為大小 n/2
- **這種忽略向下取整和向上取整的簡化通常不會影響分治遞迴解的增長階數**（Chapter 4 將討論）

### 省略基礎情況

**另一個慣例**：
- 我們將省略遞迴的基礎情況陳述
- Chapter 4 將更詳細地討論

**原因**：
- 基礎情況幾乎總是：T(n) = Θ(1) if n < n₀，對於某個常數 n₀ > 0
- 因為常數大小輸入上的演算法運行時間是常數
- 採用這個慣例可以節省大量額外的書寫

## 合併排序的分析

### 設置遞迴

**設置 T(n) 的遞迴**（n 個數字的合併排序的最壞情況運行時間）：

#### 1. 分解（Divide）

**操作**：
- 分解步驟只是計算子陣列的中間位置
- 需要常數時間

**成本**：
- D(n) = Θ(1)

#### 2. 征服（Conquer）

**操作**：
- 遞迴求解兩個子問題，每個大小為 n/2

**成本**：
- 2T(n/2)（忽略向下取整和向上取整）

#### 3. 組合（Combine）

**操作**：
- MERGE 程序在 n 元素子陣列上需要 Θ(n) 時間

**成本**：
- C(n) = Θ(n)

### 合併排序的遞迴方程

**組合成本**：
- 當我們為合併排序分析添加函數 D(n) 和 C(n) 時
- 我們添加一個 Θ(n) 的函數和一個 Θ(1) 的函數
- 這個和是 n 的線性函數
- 即當 n 很大時大致與 n 成正比
- 合併排序的分解和組合時間一起是 Θ(n)

**最終遞迴**：
- 將 Θ(n) 添加到征服步驟的 2T(n/2) 項

```
T(n) = 2T(n/2) + Θ(n)    ... (2.3)
```

### 主定理的解

**Chapter 4** 提出「主定理」（master theorem），它表明：

```
T(n) = Θ(n lg n)
```

**註釋**：
- 記號 lg n 代表 log₂ n
- 雖然對數的底數在這裡不重要
- 但作為電腦科學家，我們喜歡底數為 2 的對數
- Section 3.3 討論其他標準記號

### 與插入排序的比較

**插入排序**：
- 最壞情況運行時間：Θ(n²)

**合併排序**：
- 最壞情況運行時間：Θ(n lg n)
- 用 lg n 因子交換了 n 因子

**為什麼這是好交易？**
- 因為對數函數增長得比任何線性函數都慢
- 對於足夠大的輸入：
  - 合併排序（Θ(n lg n) 最壞情況運行時間）
  - 優於插入排序（Θ(n²) 最壞情況運行時間）

## 遞迴樹分析

### 直觀理解

**不需要主定理**：
- 我們可以直觀地理解為什麼遞迴 (2.3) 的解是 T(n) = Θ(n lg n)

**簡化假設**：
- n 是 2 的精確次方
- 隱式基礎情況是 n = 1

**簡化遞迴**：

```
T(n) = { c₁           if n = 1,
       { 2T(n/2) + c₂n if n > 1,    ... (2.4)
```

其中：
- 常數 c₁ > 0 代表解決大小為 1 的問題所需的時間
- c₂ > 0 是分解和組合步驟的每個陣列元素的時間

### 遞迴樹構造

**圖 2.5** 展示了計算遞迴 (2.4) 解的一種方法：

#### 部分 (a)：T(n)
顯示 T(n)

#### 部分 (b)：展開一層
- 將 T(n) 展開成等效的樹表示遞迴
- c₂n 項表示在遞迴頂層的分解和組合成本
- 根的兩個子樹是兩個較小的遞迴 T(n/2)

#### 部分 (c)：展開兩層
- 進一步展開 T(n/2)
- 在遞迴的第二層，每個節點的分解和組合成本為 c₂n/2

#### 部分 (d)：完全展開的樹
- 繼續展開樹中的每個節點
- 通過根據遞迴分解其組成部分
- 直到問題大小降到 1，每個成本為 c₁

### 計算總成本

**跨每層添加成本**：

1. **頂層**：
   - 總成本 = c₂n

2. **第二層**：
   - 總成本 = c₂(n/2) + c₂(n/2) = c₂n

3. **第三層**：
   - 總成本 = c₂(n/4) + c₂(n/4) + c₂(n/4) + c₂(n/4) = c₂n

**模式**：
- 每層有兩倍於上一層的節點數
- 但每個節點貢獻的成本只有上一層節點的一半
- 從一層到下一層，加倍和減半相互抵消
- 因此跨每層的成本相同：c₂n

**一般情況**：
- 在頂層下方 i 層的層有 2^i 個節點
- 每個貢獻 c₂(n/2^i) 的成本
- 頂層下方第 i 層的總成本 = 2^i × c₂(n/2^i) = c₂n

**底層**：
- 有 n 個節點，每個貢獻 c₁ 的成本
- 總成本 = c₁n

### 樹的層數

**總層數**：圖 2.5 中的遞迴樹有 **lg n + 1** 層

**證明（非正式歸納論證）**：

**基礎情況**：
- 當 n = 1 時，樹只有 1 層
- 由於 lg 1 = 0，我們有 lg n + 1 給出正確的層數

**歸納假設**：
- 假設具有 2^i 個葉子的遞迴樹的層數為 lg 2^i + 1 = i + 1
- （因為對於任何 i 值，我們有 lg 2^i = i）

**歸納步驟**：
- 由於我們假設輸入大小是 2 的精確次方
- 下一個要考慮的輸入大小是 2^(i+1)
- 具有 n = 2^(i+1) 個葉子的樹比具有 2^i 個葉子的樹多 1 層
- 因此總層數 = (i + 1) + 1 = lg 2^(i+1) + 1

### 總成本計算

**計算總成本**：
- 遞迴樹有 lg n + 1 層
- 葉子上方的每層成本為 c₂n
- 葉子層成本為 c₁n

**總成本**：

```
總成本 = c₂n × lg n + c₁n
       = c₂n lg n + c₁n
       = Θ(n lg n)
```

**結論**：遞迴 (2.4) 的解為 T(n) = Θ(n lg n)

### 2-3 霍納規則的正確性

給定多項式的係數 a₀, a₁, a₂, ..., aₙ：

```
P(x) = Σ(k=0 to n) aₖx^k
     = a₀ + a₁x + a₂x² + ... + aₙ₋₁x^(n-1) + aₙx^n
```

你想計算給定 x 值的多項式。**霍納規則**說根據此括號化計算多項式：

程序 HORNER 實現霍納規則來計算 P(x)：

```
1  p = 0
2  for i = n downto 0

4  return p
```

在第 2-3 行 for 循環的每次迭代開始時：

## 章節註釋

**Knuth (1968)**：
- 出版了三卷本《計算機程式設計的藝術》的第一卷
- 第一卷開創了對電腦演算法的現代研究，重點是運行時間的分析
- 整個系列仍然是這裡提出的許多主題的引人入勝且有價值的參考資料
- 根據 Knuth，「演算法」一詞源自「al-Khowārizmī」，一位九世紀的波斯數學家

**Aho, Hopcroft, and Ullman (1974)**：
- 提倡演算法的漸近分析
- 使用 Chapter 3 引入的記號，包括 Θ 記號
- 作為比較相對性能的手段
- 他們還普及了使用遞迴關係來描述遞迴演算法的運行時間

**Knuth 的排序演算法處理**：
- 提供了許多排序演算法的百科全書式處理
- 他對排序演算法的比較（第 381 頁）包括精確的步驟計數分析
- 就像我們在這裡對插入排序所做的那樣
- Knuth 對插入排序的討論涵蓋了演算法的幾種變體
- 其中最重要的是 Shell 排序（由 D. L. Shell 引入）
- Shell 排序對輸入的週期性子陣列使用插入排序以產生更快的排序演算法

**合併排序歷史**：
- Knuth 也描述了合併排序
- 他提到 1938 年發明了一種能夠在單次通過中合併兩副打孔卡的機械分類器
- J. von Neumann（電腦科學的先驅之一）顯然在 1945 年為 EDVAC 電腦編寫了合併排序程式

**證明程式正確的早期歷史**：
- Gries (1981) 描述，他將該領域的第一篇文章歸功於 P. Naur
- Gries 將循環不變式歸功於 R. W. Floyd
- Mitchell (1996) 的教科書是關於如何證明程式正確的良好參考
