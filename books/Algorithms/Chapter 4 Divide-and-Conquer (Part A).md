## 4.0 前言：分治法與遞迴

### 遞迴關係式（Recurrence）的基本概念

**遞迴關係式**是一個方程式或不等式，透過函數本身來描述該函數在整數或實數上的值。它包含兩種或更多種情況：

- **遞迴情況（Recursive case）**：涉及對較小輸入的遞迴呼叫
- **基底情況（Base case）**：不涉及遞迴呼叫的情況

### 演算法遞迴（Algorithmic Recurrence）

一個遞迴關係式 T(n) 稱為**演算法遞迴**，如果對於所有足夠大的閾值常數 n₀ > 0，滿足以下兩個性質：

1. **小規模輸入的界限**：對所有 n < n₀，有 T(n) = Θ(1)
2. **終止性**：對所有 n ≥ n₀，每條遞迴路徑都會在有限次遞迴後終止於定義好的基底情況

### 遞迴的慣例

**重要慣例**：當遞迴關係式沒有明確給出基底情況時，我們假設該遞迴是演算法遞迴。這意味著：
- 可以自由選擇足夠大的閾值常數 n₀ 作為基底情況的範圍（T(n) = Θ(1)）
- 大多數演算法遞迴的漸進解不依賴於閾值常數的選擇
- 可以忽略向下取整（floor）和向上取整（ceiling）符號，將整數域的遞迴轉換為實數域

### 分治法與遞迴的關係

分治法將問題分解為較小的子問題來解決。本章展示的矩陣乘法例子包括：

1. **簡單分治法**：將 n×n 矩陣乘法分解為 8 個 n/2×n/2 的子問題
   - 遞迴關係式：T(n) = 8T(n/2) + Θ(1)
   - 解：T(n) = Θ(n³)

2. **Strassen 演算法**：將問題分解為 7 個 n/2×n/2 的子問題
   - 遞迴關係式：T(n) = 7T(n/2) + Θ(n²)
   - 解：T(n) = Θ(n^(lg 7)) = O(n^2.81)

### 其他常見的遞迴模式

- **不均等分割**：T(n) = T(n/3) + T(2n/3) + Θ(n) → 解為 T(n) = Θ(n lg n)
- **線性搜尋**：T(n) = T(n-1) + Θ(1) → 解為 T(n) = Θ(n)

## 4.1 矩陣乘法

### 矩陣乘法的基本定義

給定兩個 n×n 矩陣 A = (aᵢₖ) 和 B = (bⱼₖ)，它們的乘積 C = A × B 也是一個 n×n 矩陣，其中：

```
cᵢⱼ = Σ(k=1 to n) aᵢₖ × bₖⱼ
```

### 直接方法：MATRIX-MULTIPLY

**演算法描述**：
```
MATRIX-MULTIPLY(A, B, C, n)
1  for i = 1 to n                  // 計算每一列的元素
2    for j = 1 to n                // 計算列 i 中的每個元素
3      for k = 1 to n
4        cᵢⱼ = cᵢⱼ + aᵢₖ × bₖⱼ    // 加入方程式的一項
```

**時間複雜度**：Θ(n³)
- 三層巢狀迴圈，每層執行 n 次
- 每次執行第 4 行需要常數時間

### 簡單的分治演算法

**核心思想**：將 n×n 矩陣分割為四個 n/2×n/2 的子矩陣

**矩陣分割**：
```
A = [A₁₁  A₁₂]    B = [B₁₁  B₁₂]    C = [C₁₁  C₁₂]
    [A₂₁  A₂₂]        [B₂₁  B₂₂]        [C₂₁  C₂₂]
```

**矩陣乘法公式**：
```
C₁₁ = A₁₁×B₁₁ + A₁₂×B₂₁
C₁₂ = A₁₁×B₁₂ + A₁₂×B₂₂
C₂₁ = A₂₁×B₁₁ + A₂₂×B₂₁
C₂₂ = A₂₁×B₁₂ + A₂₂×B₂₂
```

這四個方程式涉及：
- **8 次** n/2×n/2 矩陣乘法
- **4 次** n/2×n/2 矩陣加法

### 矩陣分割的實作方式

**方法一：複製法**
- 分配臨時儲存空間存放子矩陣
- 複製每個元素到對應的子矩陣位置
- 時間：Θ(n²)（需要複製 3n² 個元素）

**方法二：索引計算法**（推薦）
- 透過索引資訊指定子矩陣在原矩陣中的位置
- 不需要實際移動矩陣元素
- 時間：Θ(1)

**重要結論**：本書採用索引計算法，分割時間為 Θ(1)

### MATRIX-MULTIPLY-RECURSIVE 演算法

```
MATRIX-MULTIPLY-RECURSIVE(A, B, C, n)
1  if n == 1
2    // 基底情況
3    c₁₁ = c₁₁ + a₁₁ × b₁₁
4    return
5  // 分割
6  將 A、B、C 分割為 n/2×n/2 的子矩陣
     A₁₁, A₁₂, A₂₁, A₂₂; B₁₁, B₁₂, B₂₁, B₂₂; C₁₁, C₁₂, C₂₁, C₂₂
7  // 征服
8  MATRIX-MULTIPLY-RECURSIVE(A₁₁, B₁₁, C₁₁, n/2)
9  MATRIX-MULTIPLY-RECURSIVE(A₁₁, B₁₂, C₁₂, n/2)
10 MATRIX-MULTIPLY-RECURSIVE(A₂₁, B₁₁, C₂₁, n/2)
11 MATRIX-MULTIPLY-RECURSIVE(A₂₁, B₁₂, C₂₂, n/2)
12 MATRIX-MULTIPLY-RECURSIVE(A₁₂, B₂₁, C₁₁, n/2)
13 MATRIX-MULTIPLY-RECURSIVE(A₁₂, B₂₂, C₁₂, n/2)
14 MATRIX-MULTIPLY-RECURSIVE(A₂₂, B₂₁, C₂₁, n/2)
15 MATRIX-MULTIPLY-RECURSIVE(A₂₂, B₂₂, C₂₂, n/2)
```

### 時間複雜度分析

**遞迴關係式**：
```
T(n) = 8T(n/2) + Θ(1)
```

- 第 6 行：分割矩陣需要 Θ(1) 時間（使用索引計算）
- 第 8-15 行：8 次遞迴呼叫，每次處理 n/2×n/2 矩陣，貢獻 8T(n/2)
- 沒有合併步驟（矩陣 C 直接更新）

**解**：T(n) = Θ(n³)

**為何和直接方法一樣慢？**
- 與 merge sort 的遞迴 T(n) = 2T(n/2) + Θ(n) 相比，這裡的因子是 8 而非 2
- 遞迴樹有 8 個子節點而非 2 個，導致樹更「茂密」
- 雖然每個內部節點的成本較小（Θ(1) vs Θ(n)），但葉節點數量多得多

## 4.2 Strassen 演算法

### Strassen 演算法的突破

**歷史背景**：1969 年之前，許多數學家認為不可能在 o(n³) 時間內完成矩陣乘法。V. Strassen 發表了一個劃時代的遞迴演算法。

**核心創新**：
- 執行時間：Θ(n^(lg 7)) = O(n^2.81)
- 關鍵：使用 7 次而非 8 次子矩陣乘法
- 代價：增加更多的矩陣加法和減法（但仍是常數個）

### 減少乘法次數的直覺

**簡單例子**：計算 x² - y²

**方法一（直接）**：
- 需要 2 次乘法（x²、y²）+ 1 次減法
- 總共 3 次運算

**方法二（代數技巧）**：
```
x² - y² = (x + y)(x - y)
```
- 需要 1 次乘法 + 2 次加減法
- 總共 3 次運算，但只有 1 次乘法

**對於標量**：兩種方法差異不大
**對於大矩陣**：乘法的成本遠高於加法，方法二較優

### Strassen 演算法的四個步驟

**步驟 1：分割**
- 如果 n = 1，執行一次標量乘法和加法，時間 Θ(1)
- 否則，將 A、B、C 分割為 n/2×n/2 的子矩陣
- 時間：Θ(1)（使用索引計算）

**步驟 2：建立中間矩陣**
建立 10 個 n/2×n/2 矩陣 S₁, S₂, ..., S₁₀（每個是兩個子矩陣的和或差）：
```
S₁ = B₁₂ - B₂₂       S₆ = B₁₁ + B₂₂
S₂ = A₁₁ + A₁₂       S₇ = A₁₂ - A₂₂
S₃ = A₂₁ + A₂₂       S₈ = B₂₁ + B₂₂
S₄ = B₂₁ - B₁₁       S₉ = A₁₁ - A₂₁
S₅ = A₁₁ + A₂₂       S₁₀ = B₁₁ + B₁₂
```

建立並初始化 7 個 n/2×n/2 矩陣 P₁, P₂, ..., P₇ 來存放乘積
時間：Θ(n²)（10 次加減法 + 7 個矩陣初始化）

**步驟 3：遞迴計算 7 個矩陣乘積**
```
P₁ = A₁₁ × S₁       (= A₁₁×B₁₂ - A₁₁×B₂₂)
P₂ = S₂ × B₂₂       (= A₁₁×B₂₂ + A₁₂×B₂₂)
P₃ = S₃ × B₁₁       (= A₂₁×B₁₁ + A₂₂×B₁₁)
P₄ = A₂₂ × S₄       (= A₂₂×B₂₁ - A₂₂×B₁₁)
P₅ = S₅ × S₆        (= A₁₁×B₁₁ + A₁₁×B₂₂ + A₂₂×B₁₁ + A₂₂×B₂₂)
P₆ = S₇ × S₈        (= A₁₂×B₂₁ + A₁₂×B₂₂ - A₂₂×B₂₁ - A₂₂×B₂₂)
P₇ = S₉ × S₁₀       (= A₁₁×B₁₁ + A₁₁×B₁₂ - A₂₁×B₁₁ - A₂₁×B₁₂)
```
時間：7T(n/2)

**步驟 4：更新結果矩陣**
透過加減 P 矩陣來更新 C 的四個子矩陣：
```
C₁₁ = C₁₁ + P₅ + P₄ - P₂ + P₆
C₁₂ = C₁₂ + P₁ + P₂
C₂₁ = C₂₁ + P₃ + P₄
C₂₂ = C₂₂ + P₅ + P₁ - P₃ - P₇
```
時間：Θ(n²)（12 次加減法）

### 時間複雜度分析

**遞迴關係式**：
```
T(n) = 7T(n/2) + Θ(n²)
```

**解**：T(n) = Θ(n^(lg 7)) = O(n^2.81)

**與簡單分治法的比較**：
- 簡單方法：T(n) = 8T(n/2) + Θ(1) → Θ(n³)
- Strassen：T(n) = 7T(n/2) + Θ(n²) → Θ(n^2.81)
- 以一次遞迴乘法換取常數個子矩陣加法，獲得更好的漸進時間

**為何有效？**
雖然 Strassen 演算法每一步的工作量更大（Θ(n²) vs Θ(1)），但遞迴樹的分支因子從 8 減少到 7，使得整體漸進時間更優。

## 4.3 替代法（Substitution Method）

### 替代法的基本步驟

替代法用於求解遞迴關係式，包含兩個步驟：

1. **猜測解的形式**：使用符號常數來表示
2. **用數學歸納法證明**：證明猜測正確並求出常數

**名稱由來**：在歸納證明中，將猜測的解代入（substitute）較小值的函數

### 範例：證明 T(n) = 2T(⌊n/2⌋) + Θ(n) 的上界

**猜測**：T(n) = O(n lg n)

**歸納假設**：T(n) ≤ cn lg n，對所有 n ≥ n₀，其中 c, n₀ > 0 是待定常數

**歸納證明**：
假設對所有 n₀ ≤ k < n，T(k) ≤ ck lg k 成立。

對於 n ≥ 2n₀：
```
T(n) ≤ 2(c⌊n/2⌋ lg⌊n/2⌋) + Θ(n)
     ≤ 2(c(n/2) lg(n/2)) + Θ(n)
     = cn lg(n/2) + Θ(n)
     = cn lg n - cn lg 2 + Θ(n)
     = cn lg n - cn + Θ(n)
     ≤ cn lg n
```

最後一步成立的條件：選擇 n₀ 和 c 足夠大，使得對於 n ≥ 2n₀，cn 能夠支配 Θ(n) 項中的匿名函數。

**基底情況**：
- 選擇 n₀ = 2
- 因為 T(n) 是演算法遞迴，T(2) 和 T(3) 是常數
- 選擇 c = max{T(2), T(3)} 即可滿足 T(2) ≤ c(2 lg 2)c 和 T(3) ≤ c(3 lg 3)c

**結論**：T(n) ≤ cn lg n 對所有 n ≥ 2 成立，因此 T(n) = O(n lg n)

### 如何做出好的猜測

**方法一：類比相似的遞迴**
如果遞迴類似於之前見過的，可以猜測類似的解。

例如：T(n) = 2T(n/2 + 17) + Θ(n)
- 類似於 merge sort 的遞迴
- 當 n 很大時，n/2 和 n/2+17 的相對差異不大
- 可以猜測 T(n) = O(n lg n)

**方法二：確定鬆散的上下界**
先找出鬆散的上下界，然後縮小不確定範圍。

例如對於 T(n) = 2T(⌊n/2⌋) + Θ(n)：
- 下界：T(n) = Ω(n)（因為有 Θ(n) 項）
- 上界：T(n) = O(n²)（初始猜測）
- 逐步改進，最終收斂到正確的 T(n) = Θ(n lg n)

### 技巧：減去低階項

有時候正確猜測了緊密的漸進界，但數學歸納證明無法進行。問題通常是歸納假設不夠強。

**範例**：T(n) = 2T(n/2) + Θ(1)

**第一次嘗試**：猜測 T(n) = O(n)，假設 T(n) ≤ cn
```
T(n) ≤ 2(c(n/2)) + Θ(1)
     = cn + Θ(1)
```
❌ 無法得出 T(n) ≤ cn

**改進**：減去一個低階項，假設 T(n) ≤ cn - d，其中 d ≥ 0
```
T(n) ≤ 2(c(n/2) - d) + Θ(1)
     = cn - 2d + Θ(1)
     ≤ cn - d - (d - Θ(1))
     ≤ cn - d
```
只要選擇 d 大於 Θ-notation 中隱藏的上界常數即可 ✓

**直覺解釋**：
- 當遞迴包含多次遞迴呼叫時，加入低階項會使問題更糟（每次遞迴都加一次）
- 減去低階項則相反（每次遞迴都減一次），讓證明能夠進行

### 常見陷阱

**陷阱一：在歸納假設中使用漸進符號**

錯誤證明：
```
T(n) ≤ 2 × O(⌊n/2⌋) + Θ(n)
     = 2 × O(n) + Θ(n)
     = O(n)  ← 錯！
```

問題：O-notation 隱藏的常數在每一步都可能改變

**正確做法**：明確命名常數
```
假設 T(n) ≤ cn
T(n) ≤ 2(c⌊n/2⌋) + Θ(n)
     ≤ cn + Θ(n)
```
此時會發現 cn + Θ(n) ≠ cn（Θ(n) 的常數可能比 c 大）

**陷阱二：證明的目標與歸納假設不一致**

錯誤證明：
```
目標：證明 T(n) = O(n)
假設：T(n) ≤ cn
證明：T(n) ≤ 2(c⌊n/2⌋) + Θ(n)
           ≤ cn + Θ(n)
           = O(n)  ← 錯！
```

問題：
- 目標是證明 T(n) = O(n)
- 但歸納假設是證明 T(n) ≤ cn
- 必須明確證明 T(n) ≤ cn，而非只證明 T(n) = O(n)

**正確認知**：使用替代法或任何歸納證明時，必須證明歸納假設的精確陳述，而非只是最終目標。

## 4.4 遞迴樹方法（Recursion-Tree Method）

### 遞迴樹的基本概念

**遞迴樹**是一種視覺化工具，用於：
- 理解遞迴關係式的結構
- 產生好的猜測（用於替代法）
- 如果謹慎處理，可作為直接證明

**遞迴樹的構造**：
- 每個節點代表某個遞迴呼叫的成本
- 將每一層的成本相加，得到該層的總成本
- 將所有層的成本相加，得到總成本

### 範例：T(n) = 3T(n/4) + Θ(n²)

我們要找出 T(n) = 3T(n/4) + cn² 的上界，其中 c > 0 是 Θ(n²) 項中的上界常數。

**遞迴樹的構造過程**（參考 Figure 4.1）：

**(a) 初始狀態**：T(n)

**(b) 第一次展開**：
```
        cn²
       / | \
     T(n/4) T(n/4) T(n/4)
```

**(c) 繼續展開**：
```
           cn²
          / | \
    c(n/4)² c(n/4)² c(n/4)²
      /|\    /|\    /|\
     ... ... ... ...
```

**(d) 完全展開的樹**：
- **高度**：log₄ n（因為每次大小縮減為 n/4）
- **第 0 層**（根）：成本 = cn²
- **第 1 層**：3 個節點，每個成本 c(n/4)²，總成本 = 3 × c(n/4)² = (3/16)cn²
- **第 2 層**：9 個節點，每個成本 c(n/16)²，總成本 = 9 × c(n/16)² = (3/16)²cn²
- **第 i 層**：3ⁱ 個節點，總成本 = (3/16)ⁱcn²
- **葉節點**：3^(log₄ n) = n^(log₄ 3) 個，每個成本 Θ(1)，總成本 = Θ(n^(log₄ 3))

### 總成本計算

**各層成本的總和**：
```
T(n) = cn² + (3/16)cn² + (3/16)²cn² + ... + (3/16)^(log₄ n - 1)cn² + Θ(n^(log₄ 3))
```

**幾何級數求和**：
```
內部節點總成本 = cn² × Σ(i=0 to log₄ n - 1) (3/16)ⁱ
                = cn² × [(1 - (3/16)^(log₄ n)) / (1 - 3/16)]
                < cn² × [1 / (13/16)]
                = (16/13)cn²
```

**總時間複雜度**：
```
T(n) = (16/13)cn² + Θ(n^(log₄ 3))
```

因為 log₄ 3 ≈ 0.793 < 2，所以 n^(log₄ 3) = o(n²)

**結論**：T(n) = O(n²)

**關鍵觀察**：
- 主要成本來自樹的根部（頂層）
- 每往下一層，成本減少為前一層的 3/16
- 葉節點的總成本相對較小（低於 n²）

   - 簡單分治：T(n) = 8T(n/2) + Θ(1) → Θ(n³)
   - Strassen 演算法：T(n) = 7T(n/2) + Θ(n²) → Θ(n^2.81)

3. **解遞迴的方法**
   - **替代法**：猜測 + 數學歸納
   - **遞迴樹方法**：視覺化分析，產生猜測

### 關鍵洞察

1. **減少遞迴分支很重要**：從 8 個子問題減少到 7 個，即使增加了每層的工作量（從 Θ(1) 到 Θ(n²)），仍能獲得更好的漸進時間
2. **低階項的技巧**：在替代法中遇到困難時，嘗試減去低階項來加強歸納假設
3. **遞迴樹的威力**：可以直觀地看到每一層的成本和總成本的來源
