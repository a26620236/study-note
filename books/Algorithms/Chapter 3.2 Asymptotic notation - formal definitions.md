## 正式定義概述

漸近記號是定義在**函數**上的數學工具：
- 定義域通常是自然數集合 ℕ 或實數集合 ℝ
- 雖然我們主要用來描述演算法的運行時間，但它適用於任何函數

**重要假設**：所有在漸近記號中使用的函數都必須是**漸近非負**的（asymptotically nonnegative）：
- 函數在 n 足夠大時必須 ≥ 0

## O-notation（大 O 記號）正式定義

### 集合定義

對於給定函數 g(n)，我們定義集合 O(g(n)) 為：

```
O(g(n)) = { f(n) : 存在正常數 c 和 n₀ 使得
            0 ≤ f(n) ≤ c·g(n) 對所有 n ≥ n₀ 成立 }
```

### 直觀理解

- f(n) ∈ O(g(n)) 表示：存在某個正常數 c，使得當 n 足夠大時，f(n) 不超過 c·g(n)
- **圖形意義**（見 Figure 3.2(a)）：在 n₀ 右側，f(n) 的圖形總是在 c·g(n) 的下方或重合

### 數學證明範例

**證明**：4n² + 100n + 500 = O(n²)

**證明過程**：
1. 需要找到正常數 c 和 n₀，使得：
   ```
   4n² + 100n + 500 ≤ c·n²  對所有 n ≥ n₀
   ```

2. 兩邊除以 n²：
   ```
   4 + 100/n + 500/n² ≤ c
   ```

3. **多種選擇方式**：
   - 選擇 n₀ = 1：可以用 c = 604
   - 選擇 n₀ = 10：可以用 c = 19
   - 選擇 n₀ = 100：可以用 c = 5.05

**反例**：n³ - 100n² ≠ O(n²)

**證明**：
1. 假設 n³ - 100n² = O(n²)
2. 則存在 c 和 n₀ 使得：n³ - 100n² ≤ c·n²
3. 除以 n²：n - 100 ≤ c
4. 但當 n > c + 100 時，此不等式不成立
5. **矛盾**！所以原假設錯誤

### 記號濫用（Notation Abuse）

雖然 O(g(n)) 是一個**集合**，我們通常寫：
```
f(n) = O(g(n))  而不是  f(n) ∈ O(g(n))
```

這種記號濫用實際上有數學意義（見後面章節）。

## Ω-notation（大 Omega 記號）正式定義

### 集合定義

對於給定函數 g(n)，我們定義集合 Ω(g(n)) 為：

```
Ω(g(n)) = { f(n) : 存在正常數 c 和 n₀ 使得
            0 ≤ c·g(n) ≤ f(n) 對所有 n ≥ n₀ 成立 }
```

### 直觀理解

- f(n) ∈ Ω(g(n)) 表示：存在某個正常數 c，使得當 n 足夠大時，f(n) 至少是 c·g(n)
- **圖形意義**（見 Figure 3.2(b)）：在 n₀ 右側，f(n) 的圖形總是在 c·g(n) 的上方或重合

### 數學證明範例 1

**證明**：4n² + 100n + 500 = Ω(n²)

**證明過程**：
1. 需要找到正常數 c 和 n₀，使得：
   ```
   4n² + 100n + 500 ≥ c·n²  對所有 n ≥ n₀
   ```

2. 兩邊除以 n²：
   ```
   4 + 100/n + 500/n² ≥ c
   ```

3. **選擇**：n₀ 為任意正整數，c = 4 即可

### 數學證明範例 2

**證明**：n²/100 - 100n - 500 = Ω(n²)

**證明過程**：
1. 除以 n²：
   ```
   1/100 - 100/n - 500/n² ≥ c
   ```

2. **關鍵**：即使最高階項的係數很小（1/100），只要 n 足夠大，仍然可以找到正的 c

3. **多種選擇**：
   - n₀ = 10,005：可以選 c = 2.49 × 10⁻⁹（非常小但是正數！）
   - n₀ = 100,000：可以選 c = 0.0089
   - **n₀ 越大，c 可以越接近 1/100**

## Θ-notation（大 Theta 記號）正式定義

### 集合定義

對於給定函數 g(n)，我們定義集合 Θ(g(n)) 為：

```
Θ(g(n)) = { f(n) : 存在正常數 c₁, c₂, 和 n₀ 使得
            0 ≤ c₁·g(n) ≤ f(n) ≤ c₂·g(n) 對所有 n ≥ n₀ 成立 }
```

### 直觀理解

- f(n) ∈ Θ(g(n)) 表示：f(n) 被 g(n) **夾在中間**（在常數倍數的範圍內）
- **圖形意義**（見 Figure 3.2(c)）：在 n₀ 右側，f(n) 的圖形總是在 c₁·g(n) 和 c₂·g(n) 之間

### 關鍵性質

Θ-notation 表示**漸近緊界**：
- 同時提供上界和下界
- 兩個常數 c₁ 和 c₂ **不需要相等**

## Theorem 3.1：Θ 與 O、Ω 的關係

### 定理陳述

```
對於任意兩個函數 f(n) 和 g(n)：

f(n) = Θ(g(n))  當且僅當  f(n) = O(g(n)) 且 f(n) = Ω(g(n))
```

### 定理意義

- **Θ 是 O 和 Ω 的交集**
- 要證明緊界，可以分別證明上界和下界，然後結合起來

### 應用範例

證明 4n² + 100n + 500 = Θ(n²)：
1. 已證明：4n² + 100n + 500 = O(n²) ✓
2. 已證明：4n² + 100n + 500 = Ω(n²) ✓
3. 根據 Theorem 3.1：4n² + 100n + 500 = Θ(n²) ✓

## 漸近記號與運行時間

### 精確性原則

使用漸近記號描述運行時間時，應該：
- **盡可能精確**，但不要過度陳述（overstate）
- **明確指出適用的情況**（worst-case、best-case、all cases）

### Insertion Sort 範例

#### ✅ 正確的陳述

1. **最壞情況**：
   - Insertion Sort 的最壞情況運行時間是 **Θ(n²)**（最佳）
   - 也可以說是 O(n²) 或 Ω(n²)（正確但不夠精確）

2. **最佳情況**：
   - Insertion Sort 的最佳情況運行時間是 **Θ(n)**（最佳）
   - 也可以說是 O(n) 或 Ω(n)（正確但不夠精確）

3. **所有情況**：
   - Insertion Sort 的運行時間是 **O(n²)**（正確：所有情況都不超過 n²）
   - Insertion Sort 的運行時間是 **Ω(n)**（正確：所有情況至少是 n）

#### ❌ 錯誤的陳述

1. **過度陳述**：
   - ❌ "Insertion Sort 的運行時間是 Θ(n²)"
   - **錯誤原因**：沒有指定「最壞情況」，這是一個涵蓋所有情況的陳述，但最佳情況是 Θ(n)

2. **過度陳述下界**：
   - ❌ "Insertion Sort 的運行時間是 Θ(n)"
   - **錯誤原因**：最壞情況是 Θ(n²)，不是所有情況都是 Θ(n)

### Merge Sort 範例

**正確陳述**：
- ✅ "Merge Sort 的運行時間是 **Θ(n lg n)**"
- **原因**：Merge Sort 在**所有情況**下都是 Θ(n lg n)，不需要額外指定 worst-case 或 best-case

### 常見錯誤：混淆 O 和 Θ

❌ **錯誤說法**：
- "An O(n lg n)-time algorithm runs faster than an O(n²)-time algorithm"

**問題**：
- 所謂的 "O(n²)-time algorithm" 可能實際上是 Θ(n) 的
- O-notation 只給出上界，不代表緊界

✅ **正確做法**：
- 如果要表示緊界，使用 **Θ-notation**
- 如果只知道上界，明確說明這只是上界

### 簡潔性原則

**目標**：提供**最簡潔且最精確**的界

**範例**：某演算法在所有情況下運行時間是 3n² + 20n

- ✅ 最佳陳述：Θ(n²)
- ❌ 技術上正確但不夠好：O(n³)（不夠精確）
- ❌ 技術上正確但不夠好：Θ(3n² + 20n)（不夠簡潔，引入不必要的細節）

## 方程式中的漸近記號

### 基本規則

漸近記號在方程式中代表**匿名函數**（anonymous function）：
- 我們不在乎具體是哪個函數
- 只關心它屬於某個漸近集合

### 規則 1：右側單獨出現

```
4n² + 100n + 500 = O(n²)
```

**解釋**：等號表示**集合成員關係**（set membership）
- 等價於：4n² + 100n + 500 ∈ O(n²)

### 規則 2：出現在更大的公式中

```
2n² + 3n + 1 = 2n² + Θ(n)
```

**解釋**：Θ(n) 代表某個**匿名函數** f(n) ∈ Θ(n)
- 意思是：2n² + 3n + 1 = 2n² + f(n)，其中 f(n) = 3n + 1 ∈ Θ(n)

**用途**：消除不重要的細節，讓方程式更清晰

**範例**：Merge Sort 的遞迴關係
```
T(n) = 2T(n/2) + Θ(n)
```
- 我們不在乎精確的低階項，它們都包含在 Θ(n) 中

### 規則 3：求和表達式

```
n
Σ O(i)
i=1
```

**解釋**：只有**一個**匿名函數（關於 i 的函數）

**錯誤理解**：
```
❌ O(1) + O(2) + ... + O(n)
```
這個表達式沒有清晰的解釋（n 個不同的匿名函數）

### 規則 4：左側出現

```
2n² + Θ(n) = Θ(n²)
```

**解釋規則**：
- **無論左側的匿名函數如何選擇**，總能找到右側的匿名函數使等式成立
- 意思是：對於任意 f(n) ∈ Θ(n)，存在 g(n) ∈ Θ(n²) 使得 2n² + f(n) = g(n)

**直觀理解**：右側提供**較粗略的細節層次**

### 規則 5：鏈式關係

```
2n² + 3n + 1 = 2n² + Θ(n)
            = Θ(n²)
```

**解釋**：每個等號分別解釋
1. 第一個等號：存在 f(n) ∈ Θ(n) 使得 2n² + 3n + 1 = 2n² + f(n)
2. 第二個等號：對於任意 g(n) ∈ Θ(n)（如上面的 f(n)），存在 h(n) ∈ Θ(n²) 使得 2n² + g(n) = h(n)

**結果**：鏈式推導意味著 2n² + 3n + 1 = Θ(n²)

## 漸近記號的正確濫用

### 濫用 1：等號表示成員關係

**濫用**：寫 `f(n) = O(g(n))` 而不是 `f(n) ∈ O(g(n))`

**正當理由**：
- 有精確的數學解釋（見「方程式中的漸近記號」）
- 簡化記號，提高可讀性

### 濫用 2：從上下文推斷變數

**濫用**：O(1) 這個表達式沒有明確哪個變數趨近無窮

**範例**：
```
f(n) = O(1)
```

**解釋**：
- 從上下文知道變數是 n
- 意思是：當 n → ∞ 時，f(n) 被常數界限

**正當理由**：
- 明確寫出變數會使記號過於冗長
- 上下文通常能清楚指出變數

### 濫用 3：有限範圍內的界

**濫用**：T(n) = O(1) for n < 3

**問題**：
- 根據正式定義，O-notation 要求 n ≥ n₀
- 這個陳述關於 n < 3，似乎違反定義

**慣例解釋**：
- 意思是：存在正常數 c 使得 T(n) ≤ c 對於 n < 3
- 我們不需要為界限常數命名，保持匿名

**類似濫用**：
- T(n) = Θ(1) for n < 3
- 意思是：T(n) 在 n < 3 時被上下常數界限

### 濫用 4：部分定義的函數

**情況**：函數 f(n) 只在部分輸入上定義（例如：n 必須是 2 的冪次）

**濫用**：仍然寫 f(n) = O(g(n))

**解釋**：
- 界限 0 ≤ f(n) ≤ c·g(n) 只在 f(n) 有定義的地方成立
- 上下文通常能清楚指出這一點

### 濫用的哲學

**數學中的濫用**：
- ✅ 可以濫用記號，只要我們理解其精確含義
- ❌ 不能誤用（導致錯誤結論）
- **好處**：簡化數學語言，幫助我們專注於真正重要的事情

## o-notation 與 ω-notation

### o-notation（小 o）：非緊的上界

#### 定義

```
o(g(n)) = { f(n) : 對於任意正常數 c > 0，存在常數 n₀ > 0 使得
            0 ≤ f(n) < c·g(n) 對所有 n ≥ n₀ 成立 }
```

#### 與 O-notation 的區別

| 特性 | O-notation | o-notation |
|------|-----------|-----------|
| 不等式 | f(n) ≤ c·g(n) | f(n) < c·g(n) |
| 常數 c | **存在某個** c | **對所有** c |
| 緊界 | 可能是緊界 | **不是緊界** |

#### 極限定義

```
f(n) = o(g(n))  ⟺  lim(n→∞) f(n)/g(n) = 0
```

**直觀理解**：當 n → ∞ 時，f(n) 相對於 g(n) 變得**微不足道**（insignificant）

#### 範例

- ✅ 2n = o(n²)（正確）
  - 因為 lim(n→∞) 2n/n² = lim(n→∞) 2/n = 0

- ❌ 2n² ≠ o(n²)（錯誤）
  - 因為 lim(n→∞) 2n²/n² = 2 ≠ 0

### ω-notation（小 omega）：非緊的下界

#### 定義

```
ω(g(n)) = { f(n) : 對於任意正常數 c > 0，存在常數 n₀ > 0 使得
            0 ≤ c·g(n) < f(n) 對所有 n ≥ n₀ 成立 }
```

#### 等價定義

```
f(n) ∈ ω(g(n))  ⟺  g(n) ∈ o(f(n))
```

#### 極限定義

```
f(n) = ω(g(n))  ⟺  lim(n→∞) f(n)/g(n) = ∞
```

**直觀理解**：當 n → ∞ 時，f(n) 相對於 g(n) 變得**任意大**

#### 範例

- ✅ n²/2 = ω(n)（正確）
  - 因為 lim(n→∞) (n²/2)/n = lim(n→∞) n/2 = ∞

- ❌ n²/2 ≠ ω(n²)（錯誤）
  - 因為 lim(n→∞) (n²/2)/n² = 1/2 ≠ ∞

### 類比關係

```
o-notation 之於 O-notation  如同  ω-notation 之於 Ω-notation
```

| 記號 | 意義 | 類比 |
|------|------|------|
| O | 上界（可能緊） | ≤ |
| o | 上界（不緊） | < |
| Ω | 下界（可能緊） | ≥ |
| ω | 下界（不緊） | > |
| Θ | 緊界 | = |

## 函數比較性質

### 性質總覽

漸近記號的許多性質類似於實數的關係性質。

### 1. 遞移性（Transitivity）

對於 Θ、O、Ω、o、ω 記號都成立：

```
f(n) = Θ(g(n)) 且 g(n) = Θ(h(n))  ⟹  f(n) = Θ(h(n))
f(n) = O(g(n)) 且 g(n) = O(h(n))  ⟹  f(n) = O(h(n))
f(n) = Ω(g(n)) 且 g(n) = Ω(h(n))  ⟹  f(n) = Ω(h(n))
f(n) = o(g(n)) 且 g(n) = o(h(n))  ⟹  f(n) = o(h(n))
f(n) = ω(g(n)) 且 g(n) = ω(h(n))  ⟹  f(n) = ω(h(n))
```

### 2. 自反性（Reflexivity）

```
f(n) = Θ(f(n))
f(n) = O(f(n))
f(n) = Ω(f(n))
```

**注意**：o 和 ω 記號**不滿足**自反性（因為它們表示嚴格不等）

### 3. 對稱性（Symmetry）

```
f(n) = Θ(g(n))  ⟺  g(n) = Θ(f(n))
```

**只有 Θ 滿足對稱性**

### 4. 轉置對稱性（Transpose Symmetry）

```
f(n) = O(g(n))  ⟺  g(n) = Ω(f(n))
f(n) = o(g(n))  ⟺  g(n) = ω(f(n))
```

### 與實數的類比

| 漸近記號 | 實數關係 | 名稱 |
|----------|---------|------|
| f(n) = O(g(n)) | a ≤ b | 小於等於 |
| f(n) = Ω(g(n)) | a ≥ b | 大於等於 |
| f(n) = Θ(g(n)) | a = b | 等於 |
| f(n) = o(g(n)) | a < b | 小於 |
| f(n) = ω(g(n)) | a > b | 大於 |

### 術語

- **f(n) 漸近更小（asymptotically smaller）**：f(n) = o(g(n))
- **f(n) 漸近更大（asymptotically larger）**：f(n) = ω(g(n))

### 5. 三一律（Trichotomy）的失效

**實數的三一律**：
```
對於任意實數 a 和 b，以下恰有一個成立：
a < b  或  a = b  或  a > b
```

**函數的情況**：三一律**不成立**

**反例**：n 和 n^(1+sin n)
- sin n 在 0 和 1 之間振盪
- 指數 1 + sin n 在 1 和 2 之間振盪
- 無法用 O 或 Ω 記號比較這兩個函數

**結論**：不是所有函數都是**漸近可比較**的（asymptotically comparable）

## 重要概念總結

### 五種漸近記號對比

| 記號 | 名稱 | 界的類型 | 形式化定義 | 直觀意義 |
|------|------|---------|-----------|----------|
| **Θ** | Theta | 緊界 | c₁g(n) ≤ f(n) ≤ c₂g(n) | 夾在中間 |
| **O** | Big-O | 上界（可能緊） | f(n) ≤ c·g(n) | 不超過 |
| **Ω** | Big-Omega | 下界（可能緊） | f(n) ≥ c·g(n) | 至少 |
| **o** | Little-o | 上界（非緊） | f(n) < c·g(n) (∀c) | 嚴格小於 |
| **ω** | Little-omega | 下界（非緊） | f(n) > c·g(n) (∀c) | 嚴格大於 |

### 關鍵定理

**Theorem 3.1**：
```
f(n) = Θ(g(n))  ⟺  f(n) = O(g(n)) ∧ f(n) = Ω(g(n))
```

### 使用原則

1. **精確性**：使用最精確的記號（Θ 優於 O 或 Ω）
2. **明確性**：明確指出適用情況（worst-case、best-case、all cases）
3. **簡潔性**：提供最簡潔的界（n² 優於 3n² + 20n）
4. **上下文**：從上下文理解記號的含義（正確濫用）

### 常見錯誤

1. ❌ 混淆 O 和 Θ
2. ❌ 過度陳述運行時間範圍
