# 2026 年度規劃

## 功能開發

- 在功能開發啟動前主導可行性評估，拆解需求、評估技術風險、分配子任務，確保每個 sprint 都能在承諾的時程內交付，降低 scope creep 對整體排程的影響
- 建立並維護 Codebase 單元測試涵蓋率 100% 的標準，透過完整的測試覆蓋確保每次交付的程式碼品質可預期、可驗證，減少 regression 風險
- 從零建立 E2E 測試基礎架構，針對核心使用者流程設計測試情境，將原本仰賴人工 QA 的驗證流程轉化為可重複執行的自動化測試，在每次 release 前提供信心保障，從根本降低線上問題發生的機率

## CI/CD 基礎建設

- 建立 CI workflow 在每個 PR 自動執行 TypeScript type checking 與 ESLint linting，將程式碼品質問題攔截在 code review 之前，避免低階錯誤進入主線
- 追蹤 test coverage，在每次 CI 執行後自動產出 coverage report 並回報至 PR，讓測試覆蓋率趨勢可視化、可追蹤，協助團隊在覆蓋率下滑時即時發現
- 設計並實作 release flow 全自動化：PR 合併後自動打版本 tag，並由 GitHub Action 觸發跨環境 backport，消除人工操作失誤的空間，讓每個 release 流程標準化且可重現
- 建立 PR 通知自動化機制：PR 建立時自動推送 Slack 通知至對應頻道，同時解析 branch name 自動注入 Notion ticket 連結至 PR description，減少人工追蹤成本，提升跨部門協作透明度

## AI 輔助團隊開發

針對前端開發流程中重複性高、容易出錯的環節，設計並建立一套 Claude Code 自定義指令集，讓 AI 深度嵌入日常開發 workflow，從需求分析到 PR 合併全程輔助。

**需求分析 & 架構設計**
- `/pr-plan` — 基於 feature-spec、api-spec 與 Figma 設計稿，自動產出新功能的架構設計方案，加速技術方向的對齊與討論
- `/pr-task-breakdown` — 根據架構設計文件（`.plan.md`）自動切分實作任務，產出結構化的 task list，降低任務拆解的認知負擔

**實作 & 開發**
- `/pr-implement` — 根據任務清單（`.task.md`）逐 Phase 自動實作，確保每個階段的實作與設計文件保持一致
- `/code-explanation` — 解讀指定程式碼的邏輯、設計意圖與技術背景，必要時附上官方文件連結，協助團隊成員快速理解陌生模組

**Code Review & 交付**
- `/pr-code-review` — 比對兩個 git branch 的差異，自動產出詳細 code review 報告，提升 review 效率與一致性
- `/pr-flow` — 一鍵串接 commit → push → 建立 PR → 產生 PR description 的完整流程，將重複性操作壓縮為單一指令

## 技術升級

- 主導前端框架升級，評估 breaking changes 影響範圍、制定遷移策略並逐步推進，確保升級過程不影響既有功能穩定性，同時讓專案能跟上框架的最新效能優化與 React 19 特性
- 主導測試框架從 Jest 遷移至 Vitest，解決 ESM 相容性問題，大幅提升測試執行速度，並統一測試環境與開發環境的模組系統，降低未來維護成本

## 團隊合作

- 主動觀察團隊運作，在發現可改善的空間時提出具體建議並推動落地，包含：調整 stand up 形式提升效率、在開發前期引入技術架構討論、功能完成後留下技術文件、交付前舉辦 Demo 確保品質，以及在 Slack 新增 release channel 並統一 release version 格式，讓團隊的協作流程更透明、更標準化
- 作為前端團隊對外溝通的核心節點，在跨部門協作中主動釐清需求模糊地帶、同步技術限制與實作取捨，確保 PM、設計、後端與前端之間的資訊對齊不落差。在需求進入開發前即介入討論，將潛在的規格衝突消化在源頭，而非讓問題在實作階段才浮現，有效降低跨團隊溝通成本與返工風險

## 技術分享

- 持續關注前端生態的最新動態，並主動在內部 RD sharing 分享技術，將外部知識轉化為團隊可直接應用的洞察，獲得正向回饋
- 對外分享將 Claude Code 深度整合進工程團隊開發流程的實戰經驗：從自定義指令集的設計理念、各 skill 的使用情境，到如何讓 AI 涵蓋需求分析、架構設計、任務拆解、程式實作、code review 到 PR 交付的完整開發生命週期。將這套在實際產品開發中驗證過的 AI-assisted workflow 系統化整理，以可複製的形式分享給外部團隊，協助更多工程師將 AI 工具從輔助角色提升為開發流程的核心環節
