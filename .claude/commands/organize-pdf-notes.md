# organize-notes

整理 PDF 檔案的筆記內容。

## 工作流程

**目標 PDF 檔案**：`/Users/zonghan/Documents/study-note/books/Algorithms/Introduction to Algorithms 4th Edition-2022-english.pdf`

請按照以下步驟進行：

### 第一步：了解需求
首先詢問用戶：
- 想要整理哪個部分的內容？（例如：第幾章、哪個主題、或特定章節名稱）

### 第二步：瀏覽並確認範圍

**重要**：系統已安裝 `PyPDF2` 和 `pypdf` 套件，可以使用 Python 腳本讀取 PDF。不需要安裝額外的工具。

1. 使用 Python (`pypdf` 套件) 讀取 PDF 檔案內容
   - 先搜尋目錄找到對應章節的頁碼範圍
   - 使用 `from pypdf import PdfReader` 來讀取 PDF
2. 根據用戶提到的部分，快速瀏覽相關內容
3. 明確告知用戶：「我看到您想整理的內容在第 XX 頁到第 XX 頁，是否正確？」
4. 等待用戶確認後再繼續

### 第三步：整理筆記
確認範圍後，重新仔細閱讀指定的頁面範圍，然後以中文整理筆記。

#### 整理原則：
- **語言**：使用繁體中文
- **詳細程度**：適中
  - 不要只列出標題或關鍵字（太精簡）
  - 不要逐字翻譯或過度展開（太複雜）
  - 應該包含：主要概念、重點說明、重要例子或應用
- **格式**：
  - 使用清晰的標題層級（##、###）
  - 適當使用列表（bullet points 或編號）
  - 重要概念可以用**粗體**標示
  - 如果有程式碼或公式，保持原樣呈現
- **結構**：
  - 按照原文的邏輯順序
  - 每個主要段落或章節都應該有總結
  - 如果內容較長，在開頭加上「本節重點」

#### 範例格式：

```markdown
## [章節標題]

**本節重點**：簡短說明這部分的核心內容（1-2 句話）

### [小節標題]

主要內容說明...

- 重點一：詳細說明
- 重點二：詳細說明

**關鍵概念**：說明重要概念的定義和意義

#### 實例
如果有具體例子，在此說明...
```

### 第四步：儲存筆記
整理完成後，將筆記儲存為 Markdown 檔案：

- **檔案命名**：使用用戶提供的主題命名，格式為 `主題名稱.md`
  - 例如：用戶說要整理「排序演算法」，檔名就是 `排序演算法.md`
  - 例如：用戶說要整理「Chapter 2」，檔名就是 `Chapter 2.md`
- **儲存位置**：`/Users/zonghan/Documents/study-note/books/Algorithms/`
- 使用 Write 工具將整理好的筆記內容寫入檔案

### 注意事項
- 如果 PDF 內容很長（超過 20 頁），建議分段整理
- 保持客觀，忠於原文內容
- 如果遇到不清楚的地方，可以標註「[原文不明確]」
- 整理完成後，詢問用戶是否需要調整詳細程度或補充特定部分
