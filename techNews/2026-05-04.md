### Managed Agents: Anthropic Decouples the Brain from the Hands
### Managed Agents：Anthropic 把「大腦」與「手」拆開

[Read post](https://www.anthropic.com/engineering/managed-agents)

- **Core**
  - Architecture splits an agent into three independent pieces: brain (Claude + harness), hands (sandboxes/tools), session (event log).
  - Harness runs stateless outside the container and calls sandboxes via a tiny `execute(name, input) → string` interface.
  - Failed harnesses recover from the session log via `wake(sessionId)` / `getSession(id)` instead of losing the whole run.
- **Impact**
  - Time-to-first-token improved ~60% at p50 and >90% at p95 because sessions no longer wait on container provisioning.
  - "The harness doesn't know whether the sandbox is a container, a phone, or a Pokémon emulator" — same brain can drive any execution surface.
- **Details**
  - Credentials live outside the sandbox (Git tokens at init, OAuth via proxy), reducing prompt-injection blast radius.
-------
- **核心**
  - 把 agent 拆成三個獨立元件：大腦（Claude + harness）、手（sandbox / 工具）、session（事件日誌）。
  - Harness 跑在 container 外、無狀態，透過 `execute(name, input) → string` 這個極簡介面呼叫 sandbox。
  - Harness 掛掉時可以靠 session log 經由 `wake(sessionId)` / `getSession(id)` 接續，不必整個 run 重來。
- **影響**
  - Time-to-first-token：p50 加快約 60%、p95 加快超過 90%，因為 session 不再卡在開 container。
  - 原文：「harness 根本不在意 sandbox 是 container、手機還是 Pokémon emulator」——同一個大腦可以驅動任何執行環境。
- **細節**
  - 憑證放在 sandbox 外（Git token 在初始化階段、OAuth 透過 proxy 取用），降低 prompt injection 的影響面。

### Claude Code Auto Mode: Skipping Permissions With Two AI Classifiers
### Claude Code auto mode：用兩個 AI classifier 安全跳過權限提示

[Read post](https://www.anthropic.com/engineering/claude-code-auto-mode)

- **Core**
  - Two-layer defense: a prompt-injection probe inspects tool output before Claude sees it; a transcript classifier judges every action before it runs.
  - Tiered allowlist — file reads skip review; in-project edits bypass the classifier; shell and external actions get scrutinized.
  - Two-stage classifier: a fast filter blocks obvious cases; a slower reasoning pass handles edge cases. False positive rate dropped from 8.5% to 0.4%.
- **Impact**
  - You can run long agentic tasks without babysitting permission prompts but keep meaningful guardrails.
  - The classifier asks "did the user authorize *this specific action*", not "is the action goal-aligned" — so off-script tool calls get blocked.
- **Details**
  - Blocked actions come back to the agent with an explanation; the agent can retry within a budget instead of halting.
-------
- **核心**
  - 雙層防線：prompt injection probe 在 Claude 看到工具輸出前先掃一遍；transcript classifier 在每個動作執行前判一次。
  - 分層 allowlist——檔案讀取直接放行、專案內編輯跳過 classifier、shell 與外部動作會被審。
  - 兩階段 classifier：快篩擋明顯情況，慢推理處理邊界案例。誤判率從 8.5% 降到 0.4%。
- **影響**
  - 可以放心讓 agent 跑長任務，不必一直回應權限提示，又保有有意義的安全邊界。
  - Classifier 判斷的是「使用者有沒有授權*這個動作*」而不是「動作是否符合目標」——脫稿的 tool call 會被擋下來。
- **細節**
  - 被擋下來的動作會帶解釋回給 agent，agent 可以在預算內重試，而不是整個流程停掉。

### "HTML in Canvas" Lands in Chrome 146 (Behind a Flag)
### 「HTML in Canvas」在 Chrome 146 登場（仍在 flag 後）

[Read post](https://frontendmasters.com/blog/the-web-is-fun-again-first-experiments-with-html-in-canvas/)

- **Core**
  - New `drawElement()` API lets you render real DOM elements as canvas pixels while keeping inputs and buttons fully interactive.
  - Apply 2D Canvas, WebGL, or WebGPU shaders on top — ripples, distortions, ambient effects on real form fields.
  - Synchronize CSS transforms between DOM and canvas so visuals and hit-testing stay aligned.
- **Impact**
  - Closes a long gap: until now, "fancy WebGL effects on a real form" meant duplicating the form in canvas and faking interactivity.
  - Could enable a new class of game-like marketing pages, data-visualization overlays, and accessibility-friendly visual effects.
- **Details**
  - Chromium 146+ only, behind `chrome://flags/#canvas-draw-element`. Not production-ready; experimentation territory only.
-------
- **核心**
  - 新的 `drawElement()` API 可以把真正的 DOM 元素以 pixel 形式渲染到 canvas，input 與 button 仍然可互動。
  - 接著套 2D Canvas、WebGL、WebGPU shader——可以在真的表單欄位上做漣漪、扭曲、環境光效。
  - DOM 與 canvas 之間能同步 CSS transform，視覺與 hit-test 對齊不會跑掉。
- **影響**
  - 補上一個長年缺口：以前要在「真正表單」上加炫炮 WebGL 效果，得在 canvas 重做一次表單再假裝可互動。
  - 有機會帶出新一類像遊戲的行銷頁、資料視覺化覆蓋層，以及不破壞無障礙的視覺特效。
- **細節**
  - 僅 Chromium 146+，需在 `chrome://flags/#canvas-draw-element` 開啟，仍是實驗階段，不能上 production。

## Glossary

- **Harness (Agent Harness)**
  - The runtime layer wrapping an LLM that handles tool calls, retries, session state, and recovery — the "scaffolding" around the model.
  - 包在 LLM 外的 runtime 層，負責處理 tool call、重試、session 狀態、錯誤復原——也就是模型外面的「鷹架」。
- **TTFT (Time-To-First-Token)**
  - Latency from the moment a request is sent to the first token appearing in the response — a key UX metric for agents and chat.
  - 從送出請求到收到第一個 token 之間的延遲，是 agent 與 chat 體驗的關鍵指標。
- **Sandbox (Agent Sandbox)**
  - The isolated execution environment (often a container or VM) where an agent's shell, file edit, and code-execution tools actually run.
  - Agent 真正執行 shell、檔案編輯、跑 code 的隔離環境，通常是 container 或 VM。
- **Prompt Injection Probe**
  - A safety check that scans tool output for malicious instructions before passing it back to the model, to prevent untrusted text from hijacking the agent.
  - 在工具輸出回傳給模型前先掃過一遍，防止不受信任的文字偷塞指令給 agent 的安全檢查。
- **HNSW (Hierarchical Navigable Small World)**
  - A graph-based index used by vector databases for fast approximate-nearest-neighbor search at scale.
  - 一種圖結構索引，向量資料庫用它做大規模近似最近鄰搜尋，速度快。
- **Federated Agents**
  - Agents running across separate machines or organizations that discover, authenticate, and collaborate with each other through a shared protocol.
  - 跑在不同機器或組織的 agent，透過共同協議互相發現、驗證、協作。
- **SnapStart (AWS Lambda)**
  - A Lambda feature that snapshots an initialized JVM (or supported runtime) and restores it on cold start instead of re-initializing — cuts startup time dramatically.
  - Lambda 的功能：把已初始化的 JVM（或支援 runtime）拍快照，cold start 時用還原代替重新初始化，啟動時間大降。
- **Firecracker**
  - The lightweight micro-VM technology AWS built to run Lambda and Fargate; gives container-level density with VM-level isolation.
  - AWS 為 Lambda 與 Fargate 打造的輕量 micro-VM 技術，達成 container 等級密度與 VM 等級隔離。
- **drawElement() (HTML in Canvas)**
  - The new experimental Canvas API that draws live DOM elements as canvas pixels while preserving their interactivity.
  - 新的實驗性 Canvas API，把活生生的 DOM 元素以 pixel 形式畫到 canvas，但元素本身仍可互動。
- **Server Functions (TanStack Start)**
  - TanStack Start's RPC-style functions that run only on the server but are called from client code as if they were local — similar to Next.js Server Actions.
  - TanStack Start 的 RPC 風格函式：實際只在 server 跑，但 client 端可以像呼叫本地函式一樣呼叫——概念跟 Next.js Server Actions 相近。
