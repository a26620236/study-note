### MCP April 2026 Spec: OAuth 2.1, Tasks, and the End of SSE
### MCP 2026 年 4 月新規格：OAuth 2.1、Tasks，SSE 走入歷史

[MCP Updates Changelog](https://tokenmix.ai/blog/mcp-updates-changelog-every-protocol-change-2026)

- **Governance**
  - MCP is now governed by the Linux Foundation's Agentic AI Foundation; 10,000+ public MCP servers in the ecosystem.
- **Auth**
  - Spec now mandates OAuth 2.1 with incremental scope consent — users approve scopes as the agent actually needs them.
- **Transports**
  - SSE transport is deprecated. New servers should use **stdio for local tools** and **Streamable HTTP for remote/hosted tools**.
- **New primitives**
  - MCP Tasks (SEP-1686) and Triggers added to the spec, enabling longer-lived async work and event-driven server behavior.
-------
- **治理**
  - MCP 現由 Linux Foundation 旗下的 Agentic AI Foundation 管理；生態系內已有超過 10,000 個公開 MCP server。
- **認證**
  - 規格強制使用 OAuth 2.1 並支援增量授權範圍——使用者依 agent 實際需要逐步授權，而非一次性同意全部。
- **傳輸**
  - SSE 傳輸方式淘汰。新 server 應採用 **本地工具用 stdio**、**遠端／託管工具用 Streamable HTTP**。
- **新原語**
  - 新增 MCP Tasks（SEP-1686）與 Triggers，支援較長時間的非同步任務與事件驅動的 server 行為。

### Constructable Stylesheets — One Parse, Every Shadow Root
### Constructable Stylesheets — 一次 parse，所有 Shadow Root 共用

[Frontend Masters: Constructable Stylesheets and adoptedStyleSheets](https://frontendmasters.com/blog/constructable-stylesheets-and-adoptedstylesheets-one-parse-every-shadow-root/)

- **Core**
  - `new CSSStyleSheet()` + `adoptedStyleSheets` lets multiple shadow roots share a single parsed stylesheet.
  - Avoids the cost of re-parsing the same `<style>` block in every web component instance.
- **When relevant**
  - You're shipping a design-system web component used N times on a page and seeing CSS parse time in your perf trace.
  - Less critical for app code that isn't shadow-DOM heavy.
-------
- **核心**
  - `new CSSStyleSheet()` 搭配 `adoptedStyleSheets`，讓多個 shadow root 共用同一份解析過的樣式表。
  - 省下「同一個 `<style>` 區塊在每個 web component 實例都重新 parse 一次」的成本。
- **什麼時候有感**
  - 你在出貨一個會在頁面被用 N 次的 design-system web component，且效能 trace 看得到 CSS parse 時間。
  - 不在 shadow DOM 中重度使用的 app code 影響有限。

### Auto `sizes` on Images Lands in Firefox 150
### Firefox 150 支援圖片 `sizes="auto"`

[Frontend Masters: Auto sizes on Images](https://frontendmasters.com/blog/auto-sizes-on-images/)

- **Core**
  - `<img sizes="auto" loading="lazy" srcset="...">` lets the browser compute the correct `sizes` value automatically.
  - Removes one of the most error-prone manual fields in responsive images.
- **Caveat**
  - Only works for lazy-loaded images (`loading="lazy"`) — eager images still need an explicit `sizes`.
  - Now in Firefox 150; Chrome and Safari already shipped earlier.
-------
- **核心**
  - `<img sizes="auto" loading="lazy" srcset="...">` 讓瀏覽器自動算出正確的 `sizes` 值。
  - 拿掉 responsive image 最容易寫錯的那欄手動設定。
- **注意**
  - 只對 lazy load（`loading="lazy"`）的圖片有效；eager 圖片還是要明確 `sizes`。
  - 現已進入 Firefox 150；Chrome、Safari 之前就支援了。

## Glossary

- **MCP (Model Context Protocol)**
  - An open protocol that standardizes how LLM agents call external tools and data sources.
  - 一個開放協議，標準化 LLM agent 如何呼叫外部工具與資料來源。
- **Streamable HTTP**
  - The MCP transport replacing SSE for remote servers; supports streaming responses over a single HTTP connection.
  - MCP 用來取代 SSE 的遠端 server 傳輸方式，能在單一 HTTP 連線上串流回應。
- **OAuth 2.1 with incremental consent**
  - Auth flow where the user approves scopes as they're actually needed, rather than granting everything up front.
  - 一種授權流程：使用者在實際需要時才同意對應 scope，而不是一次性授予全部權限。
- **Agent Skill**
  - A packaged unit of capability (instructions + optional tools/files) an agent can load on demand — Claude Agent SDK's primary extension model.
  - 將「一組能力」（指示語 + 選用的工具／檔案）封裝起來、agent 可按需載入的單位，是 Claude Agent SDK 的主要擴充方式。
- **Constructable Stylesheets**
  - JS-constructed CSSStyleSheet objects that can be shared across multiple shadow roots via `adoptedStyleSheets`.
  - 用 JS 建構出的 CSSStyleSheet 物件，可透過 `adoptedStyleSheets` 在多個 shadow root 間共用。
- **Baseline (newly available)**
  - W3C status meaning a feature works across the latest versions of Chrome, Firefox, Safari, and Edge — but older versions may not.
  - W3C 的瀏覽器支援狀態，表示功能在 Chrome、Firefox、Safari、Edge 的最新版皆可用，但舊版未必。
- **Headless library**
  - A library that ships logic and state without UI — you compose your own markup and styling on top.
  - 只提供邏輯與狀態、不附 UI 的函式庫，外層 markup 與樣式由你自行組裝。
- **Parallel agents**
  - Running multiple agent instances concurrently on independent sub-tasks, then merging their outputs.
  - 同時跑多個 agent 實例處理彼此獨立的子任務，最後合併結果。
- **PID namespace isolation**
  - A Linux kernel feature that gives a process tree its own view of process IDs, preventing it from seeing or signaling host processes.
  - Linux 核心功能，讓行程樹擁有獨立的 PID 視圖，無法看到或對宿主行程發送 signal。
