# CI/CD

## 簡介

CI/CD automates building, testing, and deploying code changes. Continuous Integration merges code frequently with automated builds/tests. Continuous Delivery/Deployment extends this to staging/production. Improves software quality and development efficiency through early issue detection.

**CI/CD 的三個組成部分**

| 組成 | 說明 |
|------|------|
| **Continuous Integration (CI)** | 開發者頻繁地將代碼變更合併到共享倉庫，每次提交都觸發自動構建和測試 |
| **Continuous Delivery (CD)** | 代碼自動準備和打包，隨時可以部署到生產環境，但實際部署需要手動審批 |
| **Continuous Deployment** | 進一步自動化，當預定義條件滿足時代碼自動部署到生產環境，無需人工干預 |

**主要優勢**

| 優勢 | 說明 |
|------|------|
| **速度** | 更快地交付功能，縮短上市時間 |
| **品質** | 在開發週期更早偵測和修復錯誤 |
| **可靠性** | 較小、頻繁的變更風險更低，更易回滾 |
| **開發者體驗** | 自動化減少手動測試工作 |
| **減少倦怠** | 可預測的流程降低部署壓力 |

**有效 CI/CD 的八大基礎**
1. 單一源代碼倉庫
2. 頻繁代碼集成
3. 自動化構建
4. 自我測試構建
5. 小批量迭代
6. 穩定的與生產相匹配的測試環境
7. 最大可見性
8. 可預測的部署
