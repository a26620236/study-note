# Real Time Data

## 簡介

Real-time data is processed and delivered immediately with minimal delay for prompt system responses. Essential for financial trading, gaming, analytics, and monitoring. Uses stream processing frameworks like Apache Kafka and Flink for high-speed data flows and timely decision-making.

**什麼是 Real-time Data**

「收集後立即傳遞的信息」，時效性無延遲。通常用於導航和追蹤目的。

**核心特性**
- **即時性**：數據收集後以最小延遲到達
- **靈活性**：可以實時處理或存儲以供後續分析
- **可變性**：可以是動態（如位置等變化變量）或靜態（固定日誌條目）

**主要應用場景**
- **經濟學**：央行實時數據和官方統計數據；研究者研究實時數據誤差如何影響政策決策
- **數字廣告**：程序化廣告競拍需要「幾乎即時訪問數據」做出有效決策

#### 2. What is Real-time Data? - Qlik
> 原文：[https://www.qlik.com/us/streaming-data/real-time-data](https://www.qlik.com/us/streaming-data/real-time-data)

**實時數據示例**
- 即時股票市場價格和交易數據
- 社交媒體動態和趨勢話題
- GPS 位置追蹤和導航
- 天氣更新和警報
- 網絡監控和系統性能指標
- IoT 感測器讀數

**主要好處**
- 立即獲得洞察，基於當前情況快速決策
- 即時響應變化情況
- 在時間敏感市場中獲得競爭優勢
- 問題出現時即時識別（而非事後發現）

**關鍵技術**
- **串流處理**：Apache Kafka、Apache Flink
- **訊息佇列**：Redis、RabbitMQ
- **API 和 Webhooks**：系統間即時數據傳遞
- **NoSQL 和內存資料庫**：針對速度優化
