# GoF Design Patterns

## 簡介

Gang of Four (GoF) Design Patterns are 23 foundational software design patterns solving common object-oriented problems. Grouped into Creational (object creation), Structural (composition), and Behavioral (communication) categories. Promote code reusability, flexibility, and maintainability.

**三大分類**

**Creational Patterns（創建型）** - 關注物件創建機制

| 模式 | 說明 |
|------|------|
| Singleton | 確保類別只有一個實例 |
| Factory Method | 定義創建物件的介面，讓子類決定實例化哪個類別 |
| Abstract Factory | 創建相關物件家族而不指定具體類別 |
| Builder | 逐步構建複雜物件 |
| Prototype | 通過複製現有物件創建新物件 |

**Structural Patterns（結構型）** - 關注物件組合和關係

| 模式 | 說明 |
|------|------|
| Adapter | 使不兼容介面能夠合作 |
| Bridge | 將抽象與實現分離，各自獨立演化 |
| Composite | 將物件組合成樹形結構 |
| Decorator | 動態添加行為到物件 |
| Facade | 為複雜系統提供簡化介面 |
| Flyweight | 共享物件有效支持大量細粒度物件 |
| Proxy | 為另一物件提供代理或佔位符 |

**Behavioral Patterns（行為型）** - 關注物件交互和職責分配

| 模式 | 說明 |
|------|------|
| Chain of Responsibility | 將請求沿鏈傳遞直到被處理 |
| Command | 將請求封裝為物件 |
| Interpreter | 為語言定義語法 |
| Iterator | 順序訪問集合元素 |
| Mediator | 定義物件如何交互的物件 |
| Memento | 捕獲和恢復物件狀態 |
| Observer | 定義物件間一對多依賴關係 |
| State | 根據狀態改變物件行為 |
| Strategy | 定義一族算法，封裝每個算法 |
| Template Method | 定義算法骨架，延遲部分步驟給子類 |
| Visitor | 在不改變類別的情況下添加操作 |
