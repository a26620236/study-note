# Design and Development Principles

## 簡介

Design and development principles guide software creation. Key principles include SOLID, DRY (Don't Repeat Yourself), KISS (Keep It Simple), YAGNI (You Aren't Gonna Need It), separation of concerns, modularity, encapsulation, composition over inheritance, and loose coupling.

#### 1. Framework Design Guidelines - Microsoft
> 原文：[https://learn.microsoft.com/en-us/dotnet/standard/design-guidelines/](https://learn.microsoft.com/en-us/dotnet/standard/design-guidelines/)

.NET 框架設計指南，提供設計可擴展 .NET 的庫的準則，確保 API 一致性和易用性。指南以 `Do`、`Consider`、`Avoid`、`Do not` 為前綴組織，幫助類庫設計者理解不同解決方案之間的權衡。

**通用軟體設計原則**

| 原則 | 說明 |
|------|------|
| **SOLID** | 五大面向對象設計原則（單一職責、開閉、里氏替換、介面隔離、依賴反轉） |
| **DRY** | Don't Repeat Yourself — 避免重複代碼 |
| **KISS** | Keep It Simple, Stupid — 保持簡單 |
| **YAGNI** | You Aren't Gonna Need It — 不實現當前不需要的功能 |
| **關注點分離** | 將系統分為不同部分，每部分解決單一關注點 |
| **模組化** | 將系統分解為可獨立開發和測試的模組 |
| **封裝** | 隱藏實現細節，只暴露必要介面 |
| **組合優於繼承** | 優先使用組合而非繼承構建功能 |
| **鬆散耦合** | 最小化組件間的依賴關係 |
